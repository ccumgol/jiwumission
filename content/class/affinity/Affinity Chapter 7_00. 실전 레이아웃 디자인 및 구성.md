## 1. 레이아웃 디자인의 원칙

### 1.1 교회 주보 레이아웃의 세 가지 목표

레이아웃을 시작하기 전에 이 주보가 무엇을 달성해야 하는지를 명확히 합니다:

| 목표 | 설명 | 디자인으로 구현하는 방법 |
| --- | --- | --- |
| **정보 전달** | 예배 정보, 광고, 일정을 명확하게 | 정보 계층, 섹션 구분, 충분한 여백 |
| **브랜드 일관성** | 교회의 정체성과 분위기 반영 | 색상 팔레트, 서체, 로고 일관 사용 |
| **가독성** | 연령대에 관계없이 쉽게 읽힘 | 글꼴 크기, 줄 간격, 대비 확보 |

### 1.2 시각적 계층 (Visual Hierarchy)

독자의 시선이 중요한 정보부터 순서대로 이동하도록 계층을 설계합니다:

`시선의 흐름 (F자형 패턴, 또는 Z자형 패턴):

┌──────────────────────────────┐
│ [제목 — 가장 크고 진하게]    │ ← 1순위: 즉시 눈에 들어옴
│                              │
│ [소제목 — 중간 크기]          │ ← 2순위: 섹션 구분
│   [본문 — 작고 가독성 위주]   │ ← 3순위: 세부 내용
│   [캡션 — 가장 작게]          │ ← 4순위: 보조 정보
└──────────────────────────────┘`

**계층을 만드는 도구:**

- **크기:** 제목 > 소제목 > 본문 > 캡션
- **굵기:** Bold > Regular > Light
- **색상:** 브랜드색(강조) vs 회색(본문) vs 연회색(캡션)
- **공간:** 중요한 요소 주변에 더 많은 여백

### 1.3 여백의 중요성 (White Space)

여백은 빈 공간이 아닙니다. 여백은 시선을 안내하고, 중요한 요소를 강조하며, 페이지를 답답하지 않게 만드는 **능동적 디자인 요소**입니다.

| 여백 종류 | 설명 | 권장 |
| --- | --- | --- |
| **페이지 여백** | 인쇄 영역의 가장자리 공간 | Chapter 2 설정값 준수 |
| **요소 간 여백** | 텍스트 프레임, 이미지 사이의 공간 | 최소 0.125" |
| **섹션 간 여백** | 주제가 바뀌는 구간의 공간 | 0.25~0.375" |
| **단락 후 여백** | 단락과 단락 사이의 공간 | 스타일 Space After로 설정 |

### 1.4 시각적 균형 (Visual Balance)

페이지 전체의 시각적 무게감이 균형을 이뤄야 합니다:

- **대칭 균형:** 좌우 또는 상하가 같은 구조 → 안정감, 격식 있는 느낌
- **비대칭 균형:** 크기나 위치가 달라도 무게감이 맞춰짐 → 역동성, 현대적 느낌

> 💡 **교회 주보 권장:** 내지는 비대칭 균형, 표지는 대칭 균형이 일반적으로 효과적입니다.

## 2. 그리드 시스템 설정

### 2.1 Layout Studio의 그리드 종류

| 그리드 종류 | 설정 위치 | 용도 |
| --- | --- | --- |
| **Column Guides** | `View` → `Guides Manager` → Columns | 텍스트 단 기준선 |
| **Baseline Grid** | `View` → `Baseline Grid Settings` | 텍스트 줄 정렬 |
| **Document Grid** | `View` → `Grid and Axis` | 정밀 격자 표시 |
| **Ruler Guides** | 눈금자에서 드래그 | 개별 가이드라인 |

### 2.2 반접기 4면 주보용 그리드 설정

Statement(5.5×8.5") 문서 기준:

**Column Guides (컬럼 가이드):**

- **내지 (2~3페이지):** 2단, Gutter 0.125"
- **표지/뒷표지 (1, 4페이지):** 1단 (단 구분 없이 자유롭게)

**설정 방법:**

1. 마스터 페이지 편집 모드 진입 (내지용 B-Master)
2. `View` → `Guides Manager`
3. **Columns** 탭:
  - Number: `2`
  - Gutter: `0.125"`
4. `OK` → 마스터 편집 종료

**Baseline Grid:**

1. `View` → `Baseline Grid Settings`
2. Grid Spacing: `15pt` (본문 줄 간격 10pt × 150% = 15pt)
3. Start Position: `0.375"` (상단 여백과 일치)
4. Colour: 하늘색 (`#A0C8FF`, 연한 색으로 설정)

### 2.3 스냅 설정

1. 상단 도구 모음의 자석 아이콘 클릭 (또는 `;` 키)
2. 자석 아이콘 오른쪽 드롭다운 화살표 클릭 → Snapping Options
3. 다음 항목 체크:
  - ✅ Snap to Guides
  - ✅ Snap to Object Bounding Box
  - ✅ Snap to Margins
  - ✅ Snap to Baseline Grid (내지 작업 시)

## 3. 전체 작업 워크플로우 개요

이 챕터에서는 4면 반접기 교회 주보를 다음 순서로 완성합니다:

`STEP 1: 문서 설정 + 마스터 페이지 완성
    (Chapters 2~3 내용 적용)
        ↓
STEP 2: 1페이지 (표지) 제작
    배경 + 로고 + 예배 정보 텍스트
        ↓
STEP 3: 2~3페이지 (내지 스프레드) 제작
    성경 본문 + 설교 요약 + 광고 + 사진
        ↓
STEP 4: 4페이지 (뒷표지) 제작
    연락처 + QR코드 + 약도
        ↓
STEP 5: 전체 검토 및 세부 조정
    오탈자, 이미지, 정렬, 여백 확인
        ↓
STEP 6: Preflight 오류 점검
    → Chapter 8에서 PDF 내보내기`

## 4. STEP 1: 문서 설정 및 마스터 페이지 완성

### 4.1 새 문서 생성 (Chapter 2 복습)

`File` → `New` → Print 탭:

| 설정 항목 | 값 |
| --- | --- |
| Page Width | 5.5" |
| Page Height | 8.5" |
| DPI | 300 |
| Facing Pages | ON |
| Start Page | Right |
| Pages | 4 |
| Top/Bottom Margins | 0.375" |
| Inside Margin | 0.375" |
| Outside Margin | 0.25" |
| Bleed (전면) | 0.125" |
| Colour Format | CMYK/8 |
| Colour Profile | U.S. Web Coated (SWOP) v2 |

→ **`Create`** 클릭

### 4.2 스타일 시스템 로드 (Chapter 4 복습)

새 문서를 생성한 직후, 지난 Chapter 4에서 만들어둔 스타일 세트를 가져옵니다:

1. Text Styles 패널 → ⚙️ → **`Import Styles`**
2. 기존 주보 파일 또는 스타일 템플릿 파일 선택
3. 가져올 스타일 전체 선택 → `Import`

> 💡 처음 작업이라면 Chapter 4의 12절 스타일 세트를 직접 생성하세요.

### 4.3 마스터 페이지 완성 (Chapter 3 복습)

### A-표지 마스터 (1, 4페이지용)

**편집 진입:** Pages 패널 → A-Master 더블클릭

배치 요소:

**① 상단 배경 띠:**

- Rectangle Tool(`M`) → 전체 폭 × 1.1" 높이 드래그
- 위치: X: -0.125", Y: -0.125" (도련 포함)
- Fill: 브랜드색 (예: C:85 M:50 Y:0 K:20)
- Stroke: 없음

**② 교회 로고:**

- `File` → `Place` → 로고 PNG 파일 선택
- 배경 띠 안 왼쪽에 배치
- 권장 크기: H: 0.65", 비율 유지

**③ 바닥글 텍스트 프레임:**

- Frame Text Tool(`T`) → 페이지 하단 0.2" 위치에 프레임 생성
- 너비: 전체 폭 (0.25" 여백 기준)
- 텍스트 입력:

`  Heritage Seum Church  |  123 Main St, Palisades Park, NJ  |  heritage-seum.org`

- 스타일: `Footer Info` (Chapter 4 스타일 적용)
- 정렬: 가운데

**④ 하단 장식선:**

- Rectangle Tool → 바닥글 텍스트 바로 위에 얇은 선 생성
- H: 1pt, Fill: 브랜드색 밝은 버전 또는 흰색
- 너비: 전체 폭

**완료:** 상단 배너 **`Finish`** 클릭

### B-내지 마스터 (2~3페이지용)

A-표지 마스터를 **복제** 후 수정:

1. A-표지 오른쪽 클릭 → `Duplicate Master`
2. 이름: `B-내지`
3. B-내지 더블클릭 → 편집 모드

수정 사항:

- 상단 배경 띠 **삭제** (내지는 배경색 없음)
- 로고: **소형으로 교체** (H: 0.35" 수준, 상단 좌측 배치)
- 교회명 소형 텍스트 추가:
  - Frame Text Tool → 상단 우측에 프레임
  - 텍스트: `Heritage Seum Church` 또는 한국어명
  - 스타일: `Footer Info` (소형, 연한 색)
- **페이지 번호 추가:**
  - 왼쪽 페이지: 하단 왼쪽 → `Text` → `Insert` → `Fields` → `Page Number`
  - 오른쪽 페이지: 하단 오른쪽 → 동일 방법
  - 스타일: `Footer Info`, 가운데 정렬
- **컬럼 가이드:** `View` → `Guides Manager` → Columns: 2, Gutter: 0.125"

**완료:** `Finish` 클릭

### 마스터 페이지 적용

Pages 패널:

- 1페이지, 4페이지 → A-표지 마스터 적용
- 2~3페이지 → B-내지 마스터 적용

> (Chapter 3의 마스터 적용 방법 참조: 드래그 앤 드롭 또는 오른쪽 클릭 → `Apply Master`)

## 9. STEP 6: Preflight — 인쇄 전 오류 점검

### 9.1 Preflight란?

Preflight는 PDF 내보내기 또는 인쇄 전에 **문서의 잠재적 오류를 자동으로 점검**하는 기능입니다. 인쇄 후 발생하는 문제를 미리 발견하여 비용 손실과 재작업을 방지합니다.

### 9.2 Preflight 패널 열기

- `Window` → `Preflight`

또는:

- `View` → `Studio` → `Preflight`

### 9.3 Preflight 점검 항목

Preflight는 다음 오류와 경고를 자동으로 감지합니다:

### 🔴 오류 (Error) — 반드시 수정 필요

| 오류 유형 | 설명 | 해결 방법 |
| --- | --- | --- |
| **Missing Font** | 문서에 사용된 글꼴이 시스템에 없음 | 해당 글꼴 설치 또는 대체 폰트 지정 |
| **Missing Image** | 이미지 파일이 연결 끊김 | Resource Manager에서 `Replace`로 재연결 |
| **Overflowing Text** | 텍스트가 프레임 밖으로 넘침 | 프레임 크기 확대 또는 텍스트 줄임 |
| **Overflowing Path Text** | 경로 위 텍스트 오버플로우 | 텍스트 줄이거나 폰트 크기 축소 |

### ⚠️ 경고 (Warning) — 확인 후 판단

| 경고 유형 | 설명 | 해결 방법 |
| --- | --- | --- |
| **Low Resolution Image** | 이미지 DPI가 인쇄 권장 기준 미달 | 고해상도 원본으로 교체 |
| **Bleed Hazard** | 이미지/색상이 도련 영역에 닿지 않음 | 해당 요소를 도련 경계까지 확장 |
| **Non-Proportional Scaling** | 이미지가 비율 무시하고 늘어남 | 비율 잠금 후 재조정 |
| **Spelling Error** | 맞춤법 오류 감지 | 직접 수정 |

### 9.4 Preflight 실행 방법

1. Preflight 패널 열기
2. **`Run Preflight`** 버튼 클릭 (또는 패널이 열리면 자동 실행)
3. 오류/경고 목록 확인
4. 각 항목 클릭 → 해당 페이지/요소로 자동 이동
5. 수정 후 다시 `Run Preflight` 실행

### 9.5 오류 유형별 수정 방법 상세

### Overflowing Text 수정

1. Preflight 패널에서 해당 오류 클릭 → 해당 프레임으로 이동
2. 빨간 삼각형이 있는 프레임 확인
3. 다음 중 하나:
  - 프레임 하단 핸들 드래그하여 크기 확장
  - 텍스트 내용 편집하여 분량 줄임
  - 글꼴 크기 소폭 축소

### Low Resolution Image 수정

1. Preflight 패널에서 해당 이미지 클릭
2. Resource Manager 확인 → Placed DPI 확인
3. 해결 방법:
  - 이미지 원본이 고해상도인 경우: 이미지 표시 크기를 줄임
  - 원본 자체가 저해상도인 경우: 고해상도 이미지로 교체
  - **Canva AI (유료):** Super Resolve로 업스케일

### Bleed Hazard 수정

1. 해당 페이지로 이동
2. 배경 이미지 또는 색상 Rectangle 선택
3. X: -0.125", Y: -0.125"로 위치 설정
4. W와 H를 페이지 크기 + 0.25"로 확장
  - 예: W: 5.5" + 0.25" = 5.75", H: 8.5" + 0.25" = 8.75"

### 9.6 Preflight 자동 실행 설정

PDF 내보내기 시 자동으로 Preflight를 실행하도록 설정:

Preflight 패널 → 설정 아이콘 → **"Run Preflight on export"** 체크

이 설정을 켜두면 오류가 있는 상태에서 PDF를 내보내려 할 때 경고 대화 상자가 표시됩니다.

## 11. 용어 해설

| 용어 | 설명 |
| --- | --- |
| **Visual Hierarchy (시각적 계층)** | 독자의 시선이 중요한 정보부터 순서대로 이동하도록 크기, 굵기, 색상, 공간으로 설계하는 원칙 |
| **White Space (여백)** | 텍스트·이미지가 없는 빈 공간. 디자인의 능동적 요소로 가독성과 강조 효과를 높임 |
| **Visual Balance (시각적 균형)** | 페이지 전체의 시각적 무게감이 안정되게 분배된 상태. 대칭/비대칭 두 유형 |
| **Preflight** | 인쇄·내보내기 전 문서의 잠재적 오류(누락 이미지, 오버플로우, DPI 부족 등)를 자동 검사하는 기능 |
| **Overflowing Text** | 텍스트 내용이 프레임보다 많아 넘치는 상태. Preflight에서 오류로 표시됨 |
| **Missing Image** | 배치된 이미지 파일의 연결이 끊어진 상태. Resource Manager에서 재연결 가능 |
| **Low Resolution Image** | 인쇄 권장 DPI(300) 미달 이미지. Preflight 경고로 표시됨 |
| **Bleed Hazard** | 배경 이미지나 색상이 도련 영역에 충분히 확장되지 않은 상태 |
| **Non-Proportional Scaling** | 이미지 비율을 무시하고 변형된 상태. Preflight 경고로 표시됨 |
| **Preview Mode** | 가이드, 여백선 등을 숨기고 실제 인쇄물처럼 보여주는 보기 모드 |
| **Template (.aftemplate)** | 재사용을 위한 Affinity 문서 형식. File → New에서 불러올 수 있음 |
| **Spread View** | Facing Pages ON 상태에서 좌우 두 페이지를 나란히 보는 보기 방식 |
| **Column Guide** | 텍스트를 여러 단으로 나누는 가이드라인. 마스터 페이지에서 설정하면 전체 적용 |
| **Baseline Grid** | 여러 텍스트 프레임의 줄이 동일한 수평선에 정렬되도록 돕는 격자 |
| **Z-순서** | 레이어의 앞뒤 배치 순서. Layers 패널에서 관리 |

## 12. FAQ

**Q1. Preflight에서 오류가 없는데도 인쇄 결과물이 이상했습니다. 왜 그럴까요?**

A. Preflight는 기술적 오류(파일 누락, 오버플로우, DPI 등)는 잡아내지만, **디자인 판단의 영역**(색상 배치, 정렬 미세 오차, 텍스트 가독성 등)은 Preflight 범위 밖입니다. Preflight 통과 후에도 Preview Mode로 직접 눈으로 최종 확인하고, 가능하면 **시범 인쇄(Test Print)** 를 한 장 먼저 출력하여 확인하는 것이 가장 안전합니다.

---

**Q2. 매주 설교 제목과 날짜를 바꾸는 것 외에 반복되는 작업을 줄이는 방법이 있나요?**

A. Find & Replace(Chapter 5)를 적극 활용하세요:

- 날짜를 한 번에 교체 가능 (`5월 11일` → `5월 18일`)
- 정기적으로 바뀌는 설교자 이름, 특정 부서명도 동일하게 적용

또한, Chapter 4에서 소개한 **Data Merge** 기능(무료)을 활용하면 스프레드시트(CSV, Excel)에서 날짜, 제목, 성경 구절을 한 번에 불러와 자동으로 채울 수 있습니다. 반복되는 주보 양식이 있다면 Data Merge 활용을 강력히 권장합니다.

---

**Q3. 마스터 페이지를 A-표지와 B-내지로 나눴는데, 뒷표지(4페이지)를 표지와 완전히 다르게 만들어야 합니다. 어떻게 하나요?**

A. 선택지 두 가지:

1. **C-뒷표지** 마스터 페이지를 새로 만들어 적용 (권장)
2. 4페이지에 A-표지 마스터를 적용한 후, 다른 요소들을 Detach하여 개별 편집

뒷표지의 디자인이 표지와 크게 다르다면 별도 마스터를 만드는 것이 관리가 훨씬 쉽습니다.

---

**Q4. 스프레드(2~3페이지)에서 2페이지와 3페이지에 걸쳐 이미지를 배치하고 싶습니다.**

A. 가능합니다. Facing Pages ON 상태에서 스프레드를 넘나드는 이미지나 배경을 배치할 수 있습니다:

1. 2페이지 작업 중, 이미지 프레임을 중앙 제본선을 넘어 3페이지까지 드래그
2. 이미지를 Place하여 배치
3. 단, **인쇄 시 제본 부분에서 이미지가 잘리거나 어긋날 수 있음** — 제본선 중앙에는 중요한 내용을 배치하지 말 것
4. 실제 인쇄 전 인쇄소와 **스프레드 이미지 처리 방법** 협의 필요

---

**Q5. Preflight에서 Spelling Error가 계속 표시됩니다. 한국어 텍스트인데 영어 오류로 표시됩니다.**

A. 언어 설정이 맞지 않아서 발생하는 문제입니다:

1. 한국어 텍스트 선택
2. Character 패널 → Language: `Korean` 선택
3. 또는 해당 단락 스타일 편집 → Language: `Korean` 설정
4. 스타일 Update로 전체 적용

또는 Preflight 설정에서 Spelling 검사 항목을 일시적으로 끄는 방법도 있습니다.

---

**Q6. 완성된 주보를 인쇄소가 아니라 자체 프린터로 출력할 때 특별히 주의할 점이 있나요?**

A. 다음 사항에 주의하세요:

1. **용지 크기 확인:** 자체 프린터에서 Statement(5.5×8.5") 용지 지원 여부 확인. 미지원 시 Letter 용지에 축소 인쇄 후 반접기.
2. **양면 인쇄:** 프린터의 양면 인쇄 기능 활용. 뒤집는 방향 설정 주의 (짧은 변 뒤집기 vs 긴 변 뒤집기).
3. **색상 프로필:** 자체 프린터는 대부분 RGB로 인쇄. CMYK로 작업한 파일을 PDF로 내보낼 때 RGB 변환 옵션 고려.
4. **DPI 설정:** 자체 프린터는 300DPI 이상 설정 권장.
5. **도련 처리:** 자체 출력 시 도련 없이 Letter 용지에 인쇄 후 손으로 재단하거나, 도련 없이 설계된 버전으로 출력.

## 13. Tips: 실수를 줄이고 더 잘 사용하는 법

### 💡 Tip 1: 표지부터 시작하지 말고 마스터 페이지부터 완성하자

실전 작업에서 가장 흔한 실수는 "표지가 멋지게 나오면 내지도 빨리 되겠지"라는 생각으로 표지부터 시작하는 것입니다. 그러나 마스터 페이지의 로고나 바닥글 위치가 표지 작업 후에 바뀌면 표지도 다시 수정해야 합니다. **반드시 마스터 페이지를 먼저 완성한 후** 각 페이지 콘텐츠를 채우세요.

---

### 💡 Tip 2: 작업 중 자주 저장하자

Affinity는 자동 저장을 지원하지만, 수동 저장(`⌘ Cmd + S` / `Ctrl + S`)을 자주 하는 습관이 중요합니다. 특히 마스터 페이지 편집 완료 후, 각 페이지 완성 후, Preflight 실행 전 각 시점에 저장하세요.

---

### 💡 Tip 3: 인쇄 전 반드시 실물 크기 시범 출력하자

화면에서 완벽해 보여도 실제 인쇄물을 직접 손에 들었을 때 다르게 보일 수 있습니다. 특히:

- 글꼴 크기가 실제로는 너무 작게 느껴지는 경우
- 색상이 화면보다 어둡거나 채도가 낮게 인쇄되는 경우
- 접힌 후 내용이 접힘선에 걸리는 경우

최종 제출 전 자체 프린터로 **시범 1부 출력 후 반접기 테스트**를 반드시 진행하세요.

---

### 💡 Tip 4: 콘텐츠 수정은 별도 텍스트 파일로 관리하자

매주 텍스트 내용을 담당자가 이메일이나 메시지로 전달받는 경우가 많습니다. 해당 텍스트를 바로 Affinity에 입력하지 말고, 먼저 텍스트 에디터(메모장, Pages, Google Docs)에서 오탈자 확인 후 붙여넣기 하세요. Word나 Google Docs에서 복사한 텍스트도 Affinity에서 스타일을 다시 적용해야 하므로, 서식 없는 순수 텍스트(`Paste without formatting` 활용)로 붙여넣는 것이 편합니다.

> 🍎 **Mac:** `⌘ Cmd + Shift + V` (서식 없이 붙여넣기)
> 🖥️ **Windows:** `Ctrl + Shift + V`

---

### 💡 Tip 5: Preflight는 PDF 내보내기 직전이 아닌, 작업 중에도 수시로 실행하자

Preflight를 PDF 내보내기 직전에만 실행하면, 오류가 많을 때 수정 작업이 몰립니다. 각 페이지 완성 후 Preflight를 실행하여 오류를 조기에 발견하는 습관을 들이세요. Preflight 패널을 화면 한쪽에 열어두고 수시로 새로 고침(`Run Preflight`)하는 방식이 효율적입니다.

---

### 💡 Tip 6: 레이아웃 변경보다 콘텐츠 교체에 집중하자

매주 주보는 레이아웃이 아닌 **콘텐츠**가 바뀌는 작업입니다. 기존 레이아웃이 잘 작동하고 있다면, 매주 레이아웃을 조금씩 바꾸려는 충동을 자제하세요. 레이아웃 변경은 분기별 또는 절기별로 계획적으로 하고, 평소에는 콘텐츠 교체만 집중합니다. 일관된 레이아웃은 독자에게 신뢰감을 주고 작업 시간을 단축시킵니다.
