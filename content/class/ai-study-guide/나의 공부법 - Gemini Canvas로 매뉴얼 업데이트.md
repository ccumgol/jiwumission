---

## 1. Canvas를 ‘매뉴얼 고도화’에 쓰는 이유(핵심 기능 4가지)

- **분할 화면 협업**: 왼쪽 대화, 오른쪽 편집. 문서 흐름이 끊기지 않음
- **드래그-부분수정**: 전체 재생성 없이, 필요한 구간만 정확히 고칠 수 있음
- **퀵 툴바**: 길이/어조/검토를 빠르게 정렬해 문서 톤을 통일
- **Create(만들기)**: 본문을 기반으로 퀴즈/체크리스트/플래시카드 같은 “부록 산출물”을 빠르게 생성

---

## 2. 고도화 목표: Simple/Detail 각각 무엇을 더 좋아지게 할 것인가

### 2.1 Simple Version(Quickstart)의 업데이트 목표

- “처음 해보는 사람”이 **막힘 없이 1회 성공**하도록
- 설명보다 **실행**이 앞에 오도록
- 단계마다 **완료 기준(체크)**이 보이도록

권장 구성(섹션 템플릿)

- 목표(한 문장)
- 준비물(계정/요금제/필요한 입력)
- 클릭 경로(번호)
- 결과 확인(완료 기준)
- 자주 막히는 지점 2~3개

### 2.2 Detail Version(교육/강의용)의 업데이트 목표

- “왜 이렇게 하는지”와 “어떻게 반복 재현하는지”까지 포함
- 품질 기준(좋은 결과/나쁜 결과), 트러블슈팅, 예시 프롬프트 라이브러리 포함
- 챕터 간 용어/버튼명/표현이 흔들리지 않도록 **편집 규칙**을 둠

---

## 3. 업데이트 단위: 

이 페이지에서 편집 단위는

- 이미 만들어진 Suno 매뉴얼(Simple/Detail)의 **토글 1개 또는 소제목 1개**입니다.

> ✅ 원칙: **한 섹션씩 완성 → Notion에 반영 → 다음 섹션으로 이동**

---

## 4. Canvas로 섹션을 고도화하는 표준 절차(권장)

### STEP 1) Notion에서 ‘수정할 섹션 1개’만 복사

- Simple/Detail 중 하나에서, 고치고 싶은 토글 또는 소제목 단위를 복사합니다.

### STEP 2) Canvas에 붙여넣고 ‘섹션 증보 프롬프트’로 1차 확장

아래 프롬프트를 그대로 쓰고, 마지막에 섹션을 붙여넣습니다.

```javascript
역할: 당신은 ‘실행 매뉴얼 편집자’입니다.
목표: 아래 섹션을 “초보자가 그대로 따라하면 성공하는 수준”으로 고도화하세요.
조건:
1) 클릭 경로를 버튼/메뉴 이름 중심으로 구체화
2) 막히는 지점 3가지 + 해결법
3) 완료 기준 체크리스트 추가
4) 예시 프롬프트 3개 추가(복사해 쓰는 문장)
5) 말투/용어/표현은 기존 문서와 일관되게 유지
출력:
- 업데이트된 섹션 본문만

[여기에 섹션 붙여넣기]
```

### STEP 3) 드래그-부분수정으로 문장과 구조를 정밀 조정

다음은 “부분 수정”에 특히 잘 맞는 요청 예시입니다.

- “이 문장을 초보자도 오해하지 않게 더 구체적으로 바꿔줘.”
- “이 단계에 ‘완료 기준’을 2개만 더 추가해줘.”
- “여기 예시 프롬프트를 ‘좋은 예/나쁜 예’로 나눠줘.”
- “전후 문단의 논리 연결이 어색하니 자연스럽게 이어줘.”

### STEP 4) 퀵 툴바로 최종 정렬(톤/길이/검토)

- 섹션마다 톤이 흔들리면 문서 신뢰도가 떨어집니다.
- 섹션 단위로 ‘어조/길이/검토’를 정렬해 통일감을 만듭니다.

---

## 5. Create(만들기)로 ‘부록’을 붙여 매뉴얼을 교육용으로 확장하기

Simple/Detail을 고도화할 때 가장 효과가 큰 부록은 아래 4종입니다.

- **부록 A. 초보자 체크리스트**
- **부록 B. 자주 막히는 지점 TOP 10**
- **부록 C. 실습 퀴즈 10문항(정답/해설 포함)**
- **부록 D. 예시 프롬프트 모음(목적별)**

Create 요청 프롬프트 예시

```javascript
아래 섹션을 바탕으로 ‘자주 막히는 지점 TOP 10’을 만들어줘.
- 증상(사용자 관점)
- 원인(가능성)
- 해결 절차(번호)
- 확인 방법(완료 기준)

[여기에 섹션 붙여넣기]
```

---

## 6. 편집 규칙(일관성 유지): Simple ↔ Detail이 함께 성장하게 만들기

- **용어 사전 1개를 유지**합니다.
  - 버튼/메뉴명, 기능명, 태그 표기 등
- Simple에서 검증된 “최소 성공 경로”를 Detail의 앞부분에도 반영합니다.
- Detail에서 나온 트러블슈팅/품질 기준은 Simple에는 “요약 링크/요약 문단”으로만 내려보냅니다.
  - Simple은 끝까지 ‘짧고 확실하게’ 유지하는 것이 목표입니다.

---

## 7. 바로 복사해서 쓰는 ‘Suno 매뉴얼 업데이트 요청’ 템플릿

```javascript
다음은 Suno AI 매뉴얼의 일부 섹션입니다.
목표: 이 섹션을 더 실행 가능하게 고도화해서 Notion에 그대로 붙여넣을 수 있게 만들어줘.

필수 포함:
- 클릭 경로(버튼/메뉴 이름)
- 막히는 지점 3개 + 해결
- 완료 기준 체크리스트
- 예시 프롬프트 3개

추가로 해주면 좋은 것:
- 초보자가 자주 오해하는 표현 2개를 ‘주의’로 표시
- 같은 내용을 반복하는 문장은 제거

[여기에 섹션 붙여넣기]
```
