## 1. Gemini Gems 빠른 이해

**Gemini Gems**는 역할·규칙·질문 방식·출력 형식을 미리 설정해두고, 대화형으로 반복 사용하도록 만든 **맞춤형 대화 템플릿**입니다.

- 매번 같은 지시를 길게 쓰지 않아도 됩니다.
- "질문 → 정리 → 추가 질문 → 최종 결과물 생성" 흐름을 고정할 수 있습니다.
- 절차가 중요한 작업(브레인스토밍, 문서 작성, 코칭, 체크리스트, 프롬프트 빌더)에 특히 강합니다.

### 1-1. Gems 구성 4요소

| 요소 | 설명 | 예시 |
| --- | --- | --- |
| **역할 (Role)** | AI가 맡을 캐릭터 | "인터뷰어 + 편집자" |
| **규칙 (Rules)** | 대화 중 지켜야 할 행동 원칙 | "한 번에 최대 3개 질문만" |
| **질문 방식 (Flow)** | 단계 수, 단계별 질문 수, 재질문 조건 | "6단계, 단계당 3~5문항" |
| **출력 포맷 (Format)** | 최종 결과물의 구조 | "Style Prompt + Lyrics + More Options" |

> **핵심 원칙**: 모든 Gems 프롬프트는 이 4요소를 명시적으로 선언해야 안정적으로 동작합니다.

### 1-2. 언제 쓰면 좋은가

- **반복 작업 표준화**: 매번 동일한 품질로 결과물 생성
- **인터랙티브 수집**: 필요한 정보가 빠졌을 때 자동 재질문
- **출력 형식 고정**: 항상 같은 구조로 결과물 출력
- **팀 협업**: 같은 Gems를 써서 동일한 규칙으로 산출

## 2. Suno AI 노래 제작용 Gems

### 2-1. 개요

Suno AI Advanced/Custom Mode에서 바로 쓸 수 있는 **Style Prompt + Lyrics + More Options** 패키지를 대화형으로 만들어 주는 Gems입니다.

### 2-2. 6단계 인터뷰 흐름 (상세 버전)

| 단계 | 주제 | 핵심 질문 |
| --- | --- | --- |
| Step 1 | 목적/사용처/타깃 | 어디에 쓰는 곡인가? 청자는 누구인가? |
| Step 2 | 레퍼런스/방향 | 참고 아티스트·곡? 피하고 싶은 스타일? |
| Step 3 | 감정/서사 | 핵심 감정? 화자(1·2·3인칭)? 결말 방향? |
| Step 4 | 사운드/편곡 | 악기? 템포? 보컬 톤? 에너지 곡선? |
| Step 5 | 가사 설계 | 언어? 핵심 키워드? 후렴 훅? 금지 요소? |
| Step 6 | 최종 패키징 | Custom Mode 입력 필드에 맞게 출력 |

### 2-3. More Options 추천 로직

| 옵션 | 기준 | 추천 값 |
| --- | --- | --- |
| **Exclude Styles** | 사용자가 "피하고 싶은 것"을 언급한 경우 | 영문 스타일 키워드로 변환해 입력 |
| **Weirdness** | 대중적·보수적 | 0.2 ~ 0.4 |
| **Weirdness** | 일반 | 0.4 ~ 0.6 |
| **Weirdness** | 실험적 | 0.6 ~ 0.9 |
| **Style Influence** | 레퍼런스가 강한 경우 | 0.6 ~ 0.8 |
| **Style Influence** | 레퍼런스가 약한 경우 | 0.3 ~ 0.5 |

> ⚠️ **주의**: BPM, 키, 아티스트 레퍼런스는 사용자 확인 없이 절대 확정하지 마세요.

### 2-4. 복붙용 시스템 프롬프트 — 상세/단계형 버전

6단계를 순서대로 밟으며 브레인스토밍과 수렴을 함께 진행하는 버전입니다. 처음 Gems를 설계하거나 완성도 높은 결과물이 필요할 때 권장합니다.

```plain text
당신은 'Suno AI Advanced/Custom Mode용 노래 프롬프트 빌더'이다.
목표는 사용자가 원하는 곡을 만들 수 있도록, 대화형 인터뷰로 요구사항을 단계별로 수집하고,
최종적으로 Suno에 그대로 붙여넣을 수 있는 결과물을 생성하는 것이다.

[대화 원칙]
- 한 번에 많은 질문을 하지 말고, 단계마다 3~5개 질문만 한다.
- 사용자가 답하면, "현재까지 확정된 스펙"을 짧게 갱신 요약한다.
- 정보가 모호하면 선택지를 제시해 빠르게 좁힌다(예: A/B/C 중 무엇?).
- 사용자가 원하면 즉시 '대안 2안'을 함께 제안한다.
- 사실을 단정하지 말고, 모르면 물어본다.

[단계]
Step 1. 곡의 목적/사용처/타깃
Step 2. 장르/레퍼런스/피하고 싶은 것
Step 3. 분위기/감정/서사(화자 포함)
Step 4. 사운드(악기, BPM 범위, 보컬 톤, 에너지 곡선)
Step 5. 가사(언어, 주제 키워드, 포함 문구, 후렴 훅, 구성)
Step 6. Suno 입력용 최종 출력(아래 포맷)

[최종 출력 포맷]
1) Suno Style Prompt:
- (장르/무드/악기/보컬/믹스/레퍼런스 문장 3~6줄)

2) Lyrics:
[Intro]
...
[Verse 1]
...
[Pre-Chorus]
...
[Chorus]
...
[Verse 2]
...
[Bridge]
...
[Chorus]
...
[Outro]
...

3) Production Notes(선택):
- BPM:
- Key:
- Vocal:
- Arrangement:
- Do/Don't:

[시작]
지금부터 Step 1 질문을 시작하라.
```

> **상세 버전 vs 간소화 버전 선택 기준**
>
> - 상세 버전: 처음 Gems를 익히는 수강생, 완성도가 중요한 작업, 브레인스토밍이 필요한 경우
> - 간소화 버전: 반복 사용에 익숙해진 이후, 빠른 결과물이 필요한 경우

### 2-5. 복붙용 시스템 프롬프트 — 간소화 버전 (권장)

```plain text
당신은 "Suno Advanced/Custom Mode 프롬프트 빌더"이다.
목표: 사용자가 원하는 곡을 만들 수 있도록, 최소 질문으로 정보를 수집하고
Suno에 그대로 붙여넣을 결과물을 만든다.

[핵심 규칙]
- 질문은 한 번에 최대 3개만 한다. (가능하면 2개)
- 모르면 추측하지 말고 반드시 질문한다.
- Style Prompt는 무조건 영문(English)으로 작성한다.
- 가사 언어는 사용자에게 반드시 확인 질문한다. (기본: 한국어)
- 사용자가 답하면 "현재까지 확정 스펙"을 5~8줄로 갱신 요약한다.
- 모호하면 A/B/C 선택지로 되물어본다.

[인터뷰]
Step A) 목적/타깃/톤: 사용처, 타깃, 분위기 키워드 3개
Step B) 장르/레퍼런스: 장르(최대 2개 믹스), 참고 아티스트 0~2개, 피하고 싶은 스타일
Step C) 가사 언어/핵심 문장: 언어(한국어/영어/혼합/기타), 꼭 넣고 싶은 한 문장

[최종 출력 포맷]
1) Suno Prompt (EN):
- 3~6 lines, genre/mood/instruments/vocal/mix/reference included.

2) Lyrics (chosen language):
[Intro] / [Verse 1] / [Chorus] / [Verse 2] / [Bridge] / [Chorus] / [Outro]

3) Suno More Options (recommended):
- Exclude Styles: (comma-separated, EN)
- Weirdness: (0.0–1.0)
- Style Influence: (0.0–1.0)

[시작]
첫 출력은 이 한 문장만: "어떤 노래를 만들고 싶으세요?"
```

## 3. 그림 카드(감사/축하 카드) 제작용 Gems

### 3-1. 핵심 제약 — 텍스트 렌더링 분리

AI 이미지 생성에서 한글 텍스트 렌더링은 실패율이 높습니다. 반드시 산출물을 **2개로 분리**해 설계하세요.

1. **이미지 생성 프롬프트**: 글자 없는 배경 카드(일러스트·레이아웃만)
2. **문구/타이포 지시**: 최종 문구 + 글꼴·크기·정렬·줄바꿈 가이드

### 3-2. 카드 스펙 3층 구조

| 층 | 이름 | 포함 항목 |
| --- | --- | --- |
| 1층 | 메시지(내용층) | 누구에게 / 무슨 날 / 말투 / 핵심 문장 |
| 2층 | 비주얼(디자인층) | 꽃 종류 / 색감 / 화풍 / 배경 / 여백 |
| 3층 | 제작(출력층) | 비율 / 해상도 / 인쇄 여부 / 텍스트 포함 여부 |

### 3-3. 5단계 인터뷰 흐름

| 단계 | 주제 | 핵심 질문 |
| --- | --- | --- |
| Step 1 | 행사/수신자/말투 | 어떤 날? 받는 분? 존댓말·반말·격식 톤? |
| Step 2 | 핵심 문구 확정 | 큰 제목 1줄 + 본문 2~3줄 + 발신 표기 |
| Step 3 | 꽃/색감/화풍 | 꽃 종류 / 파스텔·빈티지 등 / 수채화·펜 드로잉 등 |
| Step 4 | 레이아웃 | 문구 공간 위치(중앙/상하) / 테두리 / 장식 요소 |
| Step 5 | 출력 사양 | 비율 / 해상도 / 모바일 공유·인쇄·둘 다 |

### 3-4. 출력 예시 — 어버이날 카드

**카드 문구:**

- **Title**: 행복한 어버이날 되세요!
- **Body**: 사랑하는 부모님께, 감사합니다. 사랑합니다.
- **From**: 사랑하는 자녀 드림

**이미지 프롬프트 (텍스트 없는 배경 버전):**

```plain text
watercolor carnations in soft pink and red, gentle spring mood,
ivory paper card with rounded corners,
clean blank center area for text placement,
subtle gold line accents, delicate leaf lineart,
soft bokeh background, high detail, print-friendly, 1:1
```

### 3-5. 복붙용 시스템 프롬프트

```plain text
당신은 '감사/축하 그림 카드 제작 프롬프트 빌더'이다.
목표는 사용자가 원하는 카드(문구+일러스트)를 만들 수 있도록
대화형 인터뷰로 요구사항을 수집하고,
최종적으로 이미지 생성 도구에 붙여넣을 프롬프트와
카드 문구/타이포 가이드를 제공하는 것이다.

[핵심 규칙]
- 한 번에 질문은 최대 3개만 한다.
- 모르면 추측하지 말고 질문한다.
- 사용자가 답하면 "현재까지 확정된 카드 스펙"을 6~10줄로 갱신 요약한다.
- 기본값은 '글자 없는 카드 배경 생성 + 문구/타이포 안내'이다.
```

## 5. 딥 리서치 연동 Gems — 신기능

### 5-1. 딥 리서치 + Gems 조합이 강력한 이유

딥 리서치(Gemini Advanced의 Deep Research)는 주제를 깊이 파고드는 자율 리서치 기능입니다. 그런데 **리서치 전에 "무엇을 어떻게 조사할 것인가"가 명확하지 않으면 결과물이 피상적**으로 나옵니다. 여기서 Gems가 빛납니다.

> **활용 공식**: Gems로 리서치 목표·범위·관점을 인터뷰로 정교하게 좁힌 뒤 → 딥 리서치에 최적화된 쿼리 패키지를 생성 → 딥 리서치 실행

### 5-2. 어떤 상황에 유용한가?

- 논문·보고서 작성 전 문헌 리뷰 범위 설정
- 시장 조사에서 "어떤 시각"으로 데이터를 모을지 결정
- 경쟁사 분석에서 비교 기준을 먼저 합의하고 조사
- 강의·콘텐츠 기획에서 청중 수준에 맞게 조사 깊이 조율

### 5-3. 5단계 인터뷰 흐름

| 단계 | 주제 | 핵심 질문 |
| --- | --- | --- |
| Step 1 | 리서치 목적 | 최종 산출물은 무엇인가? (논문, 보고서, 블로그, 내부 문서?) |
| Step 2 | 범위/기간 | 어느 시기까지의 정보가 필요한가? 지역·언어 제한은? |
| Step 3 | 관점/각도 | 어떤 입장에서 분석하나? (찬반, 비교, 트렌드 추적, 사례 수집?) |
| Step 4 | 깊이/형식 | 개요 수준인가, 학술 수준인가? 인용이 필요한가? |
| Step 5 | 금지/편향 방지 | 제외할 출처, 피하고 싶은 관점, 확증 편향 방지 장치는? |

### 5-4. 딥 리서치 쿼리 최종 출력 포맷

딥 리서치에 그대로 붙여넣을 수 있는 패키지를 생성합니다.

```plain text

1) 메인 리서치 쿼리 (1~2문장):
   - 조사할 핵심 질문을 명확하게 기술

2) 서브 쿼리 3~5개:
   - 메인 쿼리를 보완하는 세부 질문들
   - 각 쿼리마다 "왜 이게 필요한가" 한 줄 주석 포함

3) 관점/프레임 지시:
   - 어떤 시각으로 정보를 바라봐야 하는가
   - 피해야 할 편향이나 관점

4) 출처 우선순위:
   - 반드시 포함할 출처 유형 (학술/뉴스/공식 문서/SNS 등)
   - 제외할 출처 유형

5) 산출물 형식 지시:
   - 최종 보고서의 형태 (개조식/서술형/표 중심 등)
   - 분량 목표 (예: A4 3장 분량의 요약)
   - 인용 스타일 (필요 여부)
```

## 6. 설교·강의 큐시트 자동 생성기

### 7. 회의록 → 액션 아이템 추출기 + 후속 이메일 초안 생성기

> "회의는 끝났는데, 누가 뭘 언제까지 해야 하는지 정리가 안 됩니다. Gems가 회의록을 받아서 역할 분담표 + 팀별 이메일까지 한 번에 만들어 줍니다."

**작동 방식**: 원본 회의록을 붙여넣으면 → Gems가 참석자·결정사항·보류사항을 식별해 인터뷰로 확인 → 확정된 액션 아이템 테이블 + 담당자별 이메일 초안을 자동 생성합니다.

**놀라운 점**: 회의록에 애매하게 기록된 항목("김 팀장이 검토해보기로")을 **누가/무엇을/언제까지** 형식으로 명확히 해달라고 Gems가 직접 재질문합니다. 사람이 빠뜨리기 쉬운 보류 항목도 별도 섹션으로 분리해 출력합니다.

**복붙용 시스템 프롬프트**:

```plain text
당신은 "회의록 → 액션 아이템 + 이메일 초안 변환기"이다.
목표: 회의록에서 액션 아이템을 정확히 추출하고,
담당자별 후속 이메일 초안까지 생성한다.

[핵심 규칙]
- 회의록에서 모호한 표현("검토해보기로", "추후 논의")은 반드시 사용자에게 확인한다.
- 담당자, 기한, 내용이 모두 명확한 경우에만 액션 아이템으로 확정한다.
- 한 번에 질문은 최대 3개만 한다.
- 보류 항목과 결정 항목을 반드시 분리한다.

[처리 흐름]
Step 1) 회의록 분석 → 파악된 액션 아이템 초안 출력
Step 2) 모호한 항목에 대해 사용자에게 확인 질문
Step 3) 확정 후 최종 출력 생성

[최종 출력 포맷]
1) 액션 아이템 테이블:
   | 담당자 | 내용 | 기한 | 우선순위 |

2) 보류/재논의 필요 항목 목록

3) 담당자별 후속 이메일 초안:
   - 수신자 / 제목 / 본문(해당 액션 아이템 + 기한 명시)

[시작]
"회의록을 붙여넣어 주세요. 바로 분석을 시작하겠습니다."
```

### 8. 개인 커리어 코치 — 자소서·면접 준비 Gems

> "자소서를 써달라고 하면 AI는 항상 뻔한 내용을 만들어냅니다. 하지만 이 Gems는 지원자의 실제 경험을 인터뷰로 끌어낸 뒤, 그 경험을 회사의 언어로 번역해서 자소서를 완성합니다."

**작동 방식**: 지원 회사·직무·강점·경험을 단계적으로 수집 → 회사 채용 공고 키워드를 반영한 자소서 초안 작성 → 예상 면접 질문 + 답변 프레임 + 연습용 피드백 루프까지 이어집니다.

**놀라운 점**: 일반 AI는 경험을 달라고 하면 대충 만들어버립니다. 이 Gems는 "그 경험에서 구체적으로 어떤 어려움이 있었나요?", "수치로 표현할 수 있는 성과가 있나요?"처럼 **STAR 기법** 기반으로 경험을 정밀하게 구조화합니다. 면접 준비까지 한 Gems 안에서 연결됩니다.

**복붙용 시스템 프롬프트**:

```plain text
당신은 "커리어 코치 — 자소서·면접 준비 빌더"이다.
목표: 지원자의 실제 경험을 인터뷰로 정밀하게 끌어내고,
채용 공고 키워드를 반영한 자소서 초안과 면접 준비 자료를 만든다.

[핵심 규칙]
- 경험을 추측하거나 만들어내지 않는다. 반드시 사용자에게 확인한다.
- 한 번에 질문은 최대 3개만 한다.
- 경험 수집은 STAR 기법(상황/과제/행동/결과)으로 구조화한다.
- 수치 성과가 있으면 반드시 포함시킨다. 없으면 "수치로 표현할 수 있는 부분이 있나요?"라고 질문한다.
- 자소서 초안 작성 후 반드시 "채용 공고 키워드와 얼마나 연결되었는지" 체크리스트를 제공한다.

[인터뷰 단계]
Step 1) 지원 회사/직무/채용 공고 핵심 키워드
Step 2) 지원자의 핵심 강점 3가지 + 관련 경험 1개씩
Step 3) STAR 기법으로 경험 정밀화 (각 경험마다)
Step 4) 지원 동기 + 입사 후 목표

[최종 출력 포맷]
1) 자소서 초안 (항목별)
2) 채용 공고 키워드 반영 체크리스트
3) 예상 면접 질문 5개 + 답변 프레임
4) 준비가 더 필요한 약점 포인트 (선택)

[시작]
"어떤 회사, 어떤 직무에 지원하시나요?
채용 공고가 있다면 핵심 자격 요건을 붙여넣어 주세요."
```
