## GitHub이란?

**간단한 역사**
GitHub은 2008년 미국에서 시작된 서비스로, 리누스 토르발스가 만든 버전 관리 도구 **Git**을 웹에서 쉽게 쓸 수 있게 해주는 플랫폼입니다. 2018년 마이크로소프트에 인수됐고, 현재 전 세계 1억 명 이상의 개발자가 사용하는 세계 최대 코드 저장소입니다.

**역할**

- **코드 보관소**: 내 컴퓨터의 프로젝트 파일을 클라우드에 보관 (백업 효과)
- **버전 관리**: 파일을 언제, 누가, 어떻게 바꿨는지 이력이 전부 남고, 과거 시점으로 되돌릴 수 있음
- **협업 공간**: 여러 명이 같은 프로젝트를 동시에 작업
- **오픈소스 생태계**: Hugo 테마(PaperMod 등)처럼 남이 공개한 코드를 가져다 쓸 수 있음
- **자동화 연결고리**: 강좌 매뉴얼처럼 Cloudflare Pages 등이 GitHub의 변경을 감지해 자동 배포

**기본 사용 흐름**은 3단계입니다: `git add .`(변경 파일 담기) → `git commit -m "메모"`(스냅샷 기록) → `git push`(GitHub에 업로드).

## 회원가입 방법

1. https://github.com 접속 → 우측 상단 **Sign up**
2. 이메일 입력 → 비밀번호 설정 → 사용자 이름(username) 정하기
3. 이메일로 온 인증 코드 입력
4. 무료 플랜(Free) 선택하면 완료

💡 사용자 이름은 저장소 주소(`github.com/아이디/저장소명`)에 그대로 들어가니 영문 소문자로 깔끔하게 정하는 게 좋습니다. 나중에 바꿀 수는 있지만 주소가 전부 바뀌어 번거롭습니다.

## 리포지토리(저장소) 생성

1. 로그인 → 우측 상단 **+** 버튼 → **New repository**
2. Repository name 입력 (예: `myblog`)
3. **Create repository** 클릭

**생성 시 신경 쓸 옵션:**

- **Public / Private**: Public은 누구나 소스를 볼 수 있고, Private는 나만 봅니다. Hugo 블로그는 Private로 해도 Cloudflare Pages 연동과 사이트 공개가 모두 가능합니다.
- **Add a README / .gitignore / license 체크박스**: 내 컴퓨터에 이미 작업물이 있어서 올릴 경우 **전부 체크하지 마세요**. 체크하면 GitHub 쪽에 파일이 먼저 생겨서 첫 push 때 "rejected... fetch first" 충돌이 납니다. (빈 저장소에서 새로 시작하는 경우엔 README를 체크해도 무방합니다.)

## 다른 사람의 리포지토리 활용법

목적에 따라 방법이 다릅니다:

- **Clone(클론)**: 저장소를 내 컴퓨터로 통째로 복제. `git clone 주소` — 코드를 받아서 써보거나 공부할 때 기본 방법입니다.
- **Fork(포크)**: 남의 저장소를 **내 계정으로 복사**해서 내 것처럼 자유롭게 수정. 원본에 수정을 제안(Pull Request)할 때도 씁니다.
- **Submodule(서브모듈)**: 내 저장소 안에 남의 저장소를 "링크"로 포함. Hugo 테마 설치가 바로 이 방식입니다 — 원본이 업데이트되면 `git submodule update --remote`로 쉽게 따라갈 수 있습니다.
- **Download ZIP**: Git 없이 그냥 파일만 받고 싶으면 저장소 페이지의 **Code → Download ZIP**. 단, 이력 관리나 업데이트 추적은 안 됩니다.
- **Star / Watch**: 즐겨찾기와 알림 구독. 좋은 프로젝트를 나중에 다시 찾을 때 유용합니다.

⚠️ 남의 코드를 쓸 때는 저장소의 **LICENSE** 파일을 확인하세요. MIT, Apache 같은 라이선스는 자유롭게 사용 가능하지만, 라이선스가 없는 저장소는 원칙적으로 무단 사용이 안 됩니다.

프로젝트 매뉴얼의 Chapter 7과 연결해서, 실제로 저장소를 만들고 push하는 실습을 도와드릴 수도 있어요.
