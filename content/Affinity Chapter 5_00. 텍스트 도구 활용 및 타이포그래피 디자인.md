## 1. 텍스트 도구의 종류 이해하기

Layout Studio에는 두 가지 주요 텍스트 도구가 있습니다. 용도에 따라 적합한 도구를 선택해야 합니다.

### 1.1 Frame Text Tool (프레임 텍스트 도구) — 단축키 `T`

**교회 주보 작업의 핵심 도구입니다.** 드래그하여 사각형 박스를 그리면 그 안에 텍스트를 입력하는 방식입니다.

| 특징 | 설명 |
| --- | --- |
| **고정된 영역** | 지정한 박스 안에서만 텍스트가 흐름 |
| **오버플로우 관리** | 텍스트가 넘치면 빨간 삼각형으로 표시, 다음 프레임으로 연결 가능 |
| **다단 설정** | 하나의 프레임 안에 2단, 3단 컬럼 설정 가능 |
| **프레임 연결** | 여러 프레임을 이어 텍스트가 자연스럽게 흐르도록 설정 가능 |

**사용 예:** 본문 기사, 광고 섹션, 공지 텍스트, 성경 구절 블록

### 1.2 Artistic Text Tool (아트 텍스트 도구) — 단축키 `A`

클릭 한 번으로 입력을 시작하며, 프레임 없이 자유롭게 늘어나는 텍스트입니다.

| 특징 | 설명 |
| --- | --- |
| **자유 배치** | 텍스트 자체가 오브젝트처럼 동작 |
| **무한 확장** | 내용이 늘어나도 자동으로 박스가 늘어남 |
| **변형 가능** | 크기 조절, 기울이기, 원형 경로를 따라 배치 가능 |
| **연결 불가** | 다른 프레임과 텍스트 흐름 연결 불가 |

**사용 예:** 표지 제목, 로고 옆 교회 이름, 장식적 텍스트 요소

> 📌 **어떤 도구를 써야 할까요?**
>
> - 본문처럼 일정 영역에 맞춰 텍스트를 배치해야 할 때 → **Frame Text Tool**
> - 제목처럼 크기와 위치를 자유롭게 조정하는 단독 텍스트 → **Artistic Text Tool**

## 2. 텍스트 프레임 만들기와 기본 편집

### 2.1 텍스트 프레임 생성

1. 왼쪽 도구 모음에서 **Frame Text Tool** 선택 (또는 `T` 키)
2. 문서에서 **드래그**하여 원하는 크기의 프레임 생성
3. 프레임 안에 커서가 활성화되면 바로 텍스트 입력 시작

> 💡 **정확한 크기로 생성하려면:**
> 프레임 생성 후 상단 컨텍스트 도구 모음의 `W`(너비), `H`(높이) 필드에 수치 직접 입력
> 예: W: 2.5", H: 3.75"

### 2.2 텍스트 프레임 선택과 편집 모드 전환

텍스트 프레임에는 두 가지 상태가 있습니다:

| 상태 | 진입 방법 | 가능한 작업 |
| --- | --- | --- |
| **프레임 선택** (파란 테두리) | Move Tool(`V`)로 프레임 클릭 | 이동, 크기 조절, 프레임 전체 삭제 |
| **텍스트 편집** (커서 표시) | 프레임 더블클릭 또는 `T` 상태에서 클릭 | 텍스트 입력, 선택, 스타일 적용 |

> 🍎 **Mac / 🖥️ Windows 공통:**
>
> - 편집 모드 → 선택 모드: `Escape` 키
> - 선택 모드 → 편집 모드: `Enter` 키 또는 더블클릭

### 2.3 텍스트 선택 방법

| 목적 | 방법 |
| --- | --- |
| 단어 선택 | 단어 더블클릭 |
| 단락 전체 선택 | 단락 내 트리플 클릭 |
| 프레임 내 전체 텍스트 | `⌘ Cmd + A` / `Ctrl + A` |
| 줄 끝까지 선택 | `Shift + End` |
| 단락 끝까지 선택 | `Shift + ↓` |

### 2.4 텍스트 프레임 크기 조절

- **Move Tool**로 프레임 선택 후 핸들 드래그
- `Shift` 누른 채 드래그: 비율 유지 (프레임은 보통 필요 없음)
- 컨텍스트 도구 모음 W/H 필드에 수치 입력: 정확한 크기 설정

### 2.5 텍스트 프레임 위치 이동

- Move Tool로 프레임 선택 → 드래그
- 컨텍스트 도구 모음 X/Y 필드에 수치 입력: 정확한 좌표 지정
  - X: 페이지 왼쪽 가장자리에서의 거리
  - Y: 페이지 위쪽 가장자리에서의 거리

## 3. Text Frame 패널 — 프레임 내부 설정

Text Frame 패널은 텍스트 프레임 자체의 레이아웃을 제어합니다. 기본적으로 숨겨져 있으므로 별도로 열어야 합니다.

**열기 경로:**

- `Window` → `Text` → `Text Frame`

### 3.1 주요 설정 항목

### 내부 여백 (Insets)

프레임 테두리와 텍스트 사이의 여백입니다. 텍스트가 프레임 가장자리에 너무 붙지 않도록 조정합니다.

- Top / Bottom / Left / Right 각각 설정 가능
- 자물쇠 아이콘으로 전체 연동
- **권장값:** 프레임에 배경색이 있을 경우 0.1~0.15" 설정

### 컬럼 (Columns)

하나의 프레임 안에서 텍스트를 여러 단으로 나눕니다.

- **Number (단 수):** 1, 2, 3, 4… (기본값: 1)
- **Gutter (단 간격):** 단과 단 사이의 공간 (권장: 0.125~0.2")
- **Balance Columns:** 마지막 줄이 짧을 때 단별 텍스트 양을 균등하게 분배

`[단 수: 2, Gutter: 0.125"]

┌──────────┬──────────┐
│ 1단      │ 2단      │
│ 텍스트   │ 텍스트   │
│ 내용     │ 내용     │
│ …        │ …        │
└──────────┴──────────┘`

### 수직 정렬 (Vertical Alignment)

프레임 내에서 텍스트의 수직 위치를 설정합니다.

| 옵션 | 설명 | 사용 예 |
| --- | --- | --- |
| **Top** | 텍스트가 상단부터 채워짐 (기본값) | 일반 본문 |
| **Center** | 텍스트가 프레임 중앙에 배치 | 제목 박스, 캡션 박스 |
| **Bottom** | 텍스트가 하단에 붙음 | 특수 레이아웃 |
| **Justify** | 텍스트가 상하 공간을 균등하게 채움 | 고정 높이 컬럼 |

### 기준선 그리드 (Use Baseline Grid)

이 체크박스를 켜면 해당 프레임의 텍스트가 기준선 그리드에 스냅됩니다. 자세한 내용은 [섹션 9](https://www.notion.so/ccumgol/Affinity-Studio-2a3091c284f6803fb17af451ba3fbdaf?p=364091c284f680e3a72ed4a2c33e6f11&pm=s#9-%EA%B8%B0%EC%A4%80%EC%84%A0-%EA%B7%B8%EB%A6%AC%EB%93%9C-baseline-grid-%ED%99%9C%EC%9A%A9)에서 다룹니다.

## 4. 텍스트 흐름 연결 (Linking Text Frames)

긴 기사나 광고 내용이 한 프레임에 다 들어가지 않을 때, 여러 프레임을 연결하여 텍스트가 자연스럽게 이어지도록 할 수 있습니다.

### 4.1 텍스트 흐름 연결의 원리

`[프레임 1] ──텍스트 흐름──→ [프레임 2] ──텍스트 흐름──→ [프레임 3]
"…부활하신 주님을       "그러므로 우리는        "이 모든 것이
 믿는 우리는…"           함께 모여…"             가능한 이유는…"

       ▲                         ▲
   삼각형 아이콘              삼각형 아이콘
  (흰색: 연결됨)             (흰색: 연결됨)`

### 4.2 텍스트 프레임 연결 방법

**방법 1 — 연결 아이콘 클릭:**

1. Move Tool(`V`)로 연결할 **첫 번째 프레임** 선택
2. 프레임 **오른쪽 하단의 삼각형 아이콘** (Flow Out) 클릭
  - 빨간 삼각형: 오버플로우 있음
  - 흰색 삼각형: 연결 가능 상태
3. 커서가 텍스트 연결 모드로 변경됨 (사슬 아이콘)
4. **두 번째 프레임을 클릭** → 연결 완료

**방법 2 — 새 프레임을 바로 그리기:**

1. 첫 번째 프레임 오른쪽 하단 삼각형 클릭
2. 빈 공간에서 **드래그** → 두 번째 프레임이 바로 생성되고 연결됨

**방법 3 — 메뉴:**

- `Text` → `Link Text Frames`

### 4.3 연결 해제

1. 연결을 끊을 프레임의 연결 아이콘 클릭
2. `Text` → `Unlink Text Frame` 선택

> ⚠️ 연결을 끊으면 두 번째 프레임 이후의 텍스트는 그 프레임 안에 머물고, 첫 번째 프레임의 오버플로우 상태로 돌아가지 않습니다.

### 4.4 연결 상태 시각적 확인

- **View** → **Show Text Flow** 활성화 시 프레임 간 연결 화살표가 표시됨
- 연결된 프레임들은 Layers 패널에서 같은 "Story" 그룹으로 묶여 표시됨

## 5. 오버플로우와 AutoFlow 처리

### 5.1 오버플로우란?

텍스트 내용이 프레임보다 많을 때 발생합니다. 프레임 하단에 **빨간 삼각형**이 표시됩니다.

`┌──────────────────┐
│ 본문 텍스트가    │
│ 여기까지만 보임  │
│                  │
└────────── [▼] ──┘  ← 빨간 삼각형 = 오버플로우`

오버플로우 텍스트는 사라진 것이 아니라 **보이지 않는 영역에 저장**되어 있습니다. 프레임을 늘리거나 다음 프레임으로 연결하면 다시 표시됩니다.

### 5.2 오버플로우 처리 방법

**방법 1 — 프레임 크기 늘리기:**
Move Tool로 프레임 선택 → 아래 핸들 드래그로 높이 확장

**방법 2 — 다음 프레임으로 연결 (섹션 4 참조):**
빨간 삼각형 클릭 → 연결 프레임 지정

**방법 3 — AutoFlow 사용:**

AutoFlow는 오버플로우 텍스트를 **자동으로 새 페이지와 프레임에 배분**하는 기능입니다.

1. 오버플로우 상태의 프레임 삼각형 클릭 (커서가 연결 모드로 변경됨)
2. 다음 중 하나 선택:

| 단축키 조합 | 결과 |
| --- | --- |
| 그냥 클릭 (빈 공간) | 클릭한 위치에 단일 프레임 생성 후 연결 |
| `⌘ Cmd + 클릭` (Mac) / `Ctrl + 클릭` (Win) | 한 페이지에 동일 크기 프레임 1개 자동 생성 |
| `⌘ Cmd + Shift + 클릭` (Mac) / `Ctrl + Shift + 클릭` (Win) | 텍스트가 끝날 때까지 여러 페이지에 프레임 자동 생성 |

> 💡 AutoFlow는 **일회성 작업**입니다. 이후 텍스트를 추가하거나 프레임 크기를 줄이면 다시 오버플로우가 발생합니다. 이 경우 다시 AutoFlow를 실행하거나 수동으로 처리합니다.

## 6. 텍스트 래핑 (Text Wrapping) — 이미지 주변 텍스트 배치

텍스트 래핑은 이미지나 도형 옆에 텍스트가 자연스럽게 흐르도록 설정하는 기능입니다.

### 6.1 Text Wrapping 패널 열기

- `View` → `Studio` → `Text Wrapping`
- 또는 이미지 선택 후 상단 컨텍스트 도구 모음의 래핑 아이콘 클릭

### 6.2 래핑 스타일 종류

| 스타일 아이콘 | 이름 | 설명 |
| --- | --- | --- |
| 없음 | **None** | 래핑 없음. 텍스트와 이미지 겹침 |
| 사각형 | **Square** | 이미지 경계 박스(사각형) 주변으로 텍스트 흐름 |
| 투명 영역 | **Tight** | 이미지의 실제 윤곽선(투명 영역 제외)을 따라 텍스트 흐름 |
| 상하만 | **Above & Below** | 이미지 위아래로만 텍스트 배치, 좌우 없음 |
| 왼쪽만 | **Left** | 이미지 왼쪽에만 텍스트 배치 |
| 오른쪽만 | **Right** | 이미지 오른쪽에만 텍스트 배치 |

### 6.3 래핑 간격 (Wrap Distance) 설정

이미지와 텍스트 사이의 여백을 설정합니다:

- Top / Bottom / Left / Right 각각 설정
- **권장값:** 0.1~0.15" (균등 적용)

### 6.4 텍스트 래핑 적용 단계

1. 래핑할 **이미지 또는 도형** 선택 (Move Tool)
2. Text Wrapping 패널에서 **래핑 스타일** 선택
3. **Distance** 필드에 여백 값 입력
4. 이미지를 텍스트 프레임 위로 이동 → 자동으로 텍스트가 이미지를 피해 흐름

> 💡 **Tight 래핑 사용 시 팁:**
> PNG처럼 투명 배경이 있는 이미지에서 가장 효과적입니다. 인물 사진 등에 적용하면 이미지 실루엣 주변으로 텍스트가 흐릅니다.

## 7. Character 패널 — 글꼴 세부 설정

Character 패널은 선택된 텍스트의 글꼴 속성을 세밀하게 제어합니다.

**열기 경로:**

- `View` → `Studio` → `Character`

### 7.1 주요 항목

### 글꼴 선택 (Font Family / Style)

- **Font Family:** 글꼴 종류 선택 (예: Noto Serif KR, Playfair Display)
- **Font Style:** Regular / Bold / Italic / SemiBold 등 서체 스타일
- **Size:** 글꼴 크기 (pt 단위)

> 💡 **한국어 글꼴 설치 확인:**
> 시스템에 설치된 글꼴만 목록에 표시됩니다. Noto Serif KR / Noto Sans KR은 Google Fonts에서 무료 다운로드 후 시스템에 설치하면 Affinity에서 바로 사용 가능합니다.
>
> - 🍎 **Mac:** 글꼴 파일 더블클릭 → "Font Book에서 열기" → 설치
> - 🖥️ **Windows:** 글꼴 파일 오른쪽 클릭 → "설치"

### 색상 (Colour)

텍스트 색상을 설정합니다. CMYK 색상 모드에서는 CMYK 값으로 지정합니다.

> 📌 **CMYK 색상 권장값 (교회 주보):**
>
> - 본문 검정: C:0 M:0 Y:0 K:100 (Rich Black 아닌 순수 검정 사용)
> - 제목 브랜드 색: 인쇄소 제출 전 색상 프루프 확인 권장

### 자간: 트래킹 (Tracking) vs 커닝 (Kerning)

| 항목 | 적용 범위 | 용도 |
| --- | --- | --- |
| **Tracking (자간)** | 선택된 전체 텍스트 | 제목의 글자 간격 전체 조정 |
| **Kerning (커닝)** | 커서가 위치한 두 글자 사이 | 특정 두 글자의 간격 미세 조정 |

- Tracking 값: 양수 → 간격 넓어짐, 음수 → 간격 좁아짐 (단위: 1/1000em)
- 제목 텍스트에서 All Caps 적용 시: Tracking +50~100으로 가독성 향상

### 기준선 이동 (Baseline Shift)

텍스트를 기준선보다 위(양수)나 아래(음수)로 이동합니다.

- 사용 예: 윗첨자(위첨자), 아랫첨자 표기, 장식적 효과

### 언어 (Language)

- 선택된 텍스트의 언어 설정
- 맞춤법 검사, 하이픈 처리에 영향

## 8. Paragraph 패널 — 단락 세부 설정

Paragraph 패널은 단락 수준의 정렬, 간격, 들여쓰기를 세밀하게 제어합니다.

**열기 경로:**

- `View` → `Studio` → `Paragraph`

### 8.1 정렬 (Alignment)

| 아이콘 | 이름 | 사용 예 |
| --- | --- | --- |
| ◀━━ | **Left** (왼쪽 정렬) | 짧은 공지 텍스트 |
| ━◆━ | **Center** (가운데 정렬) | 제목, 캡션 |
| ━━▶ | **Right** (오른쪽 정렬) | 날짜, 페이지 번호 |
| ◀━━▶ | **Justify** (양쪽 정렬) | 본문 기사 (가장 일반적) |
| ◀━━\| | **Justify Last Line Left** | 본문 (마지막 줄 왼쪽 정렬 — 권장) |

> 💡 **한국어 본문 권장 정렬: Justify (양쪽 정렬) + Last Line Left**
> 양쪽 정렬은 깔끔한 단 경계를 만들지만, 마지막 줄까지 강제 정렬되면 글자 간격이 이상해질 수 있습니다. `Justify Last Line Left`가 이를 방지합니다.

### 8.2 줄 간격 (Leading)

- **Auto (자동):** 글꼴 크기의 약 120%로 자동 설정
- **수동 입력:** pt 단위로 직접 지정 (예: 10pt 글꼴 → 15~17pt Leading)
- 한국어 본문: 글꼴 크기의 150~170% 권장

> 💡 **줄 간격 계산 예시:**
>
> - 10pt 본문 → Leading: 15~17pt
> - 13pt 소제목 → Leading: 15~16pt
> - 18pt 제목 → Leading: 18~20pt (제목은 줄 간격을 좁게)

### 8.3 들여쓰기 (Indents)

| 항목 | 설명 | 권장값 |
| --- | --- | --- |
| **First Line** | 첫 줄만 들여쓰기 | 0.2~0.25" |
| **Left Indent** | 단락 전체 왼쪽 들여쓰기 | 인용문: 0.25" |
| **Right Indent** | 단락 전체 오른쪽 들여쓰기 | 인용문: 0.25" |

> 📌 **첫 줄 들여쓰기 vs 단락 후 공백:**
>
> - 책이나 정식 인쇄물: 첫 줄 들여쓰기(First Line Indent) + 단락 후 공백 없음
> - 주보, 뉴스레터 스타일: 들여쓰기 없음 + 단락 후 공백(Space After) 4~6pt
> 두 방식을 혼용하지 않는 것이 원칙입니다.

### 8.4 단락 앞/뒤 간격 (Space Before / After)

단락과 단락 사이의 수직 여백입니다. `Enter` 키를 여러 번 눌러 줄을 띄우는 것보다 이 값으로 간격을 제어하는 것이 전문적입니다.

- **Space After Body (본문):** 4~6pt
- **Space After Heading (제목):** 6~10pt
- **Space Before Heading:** 10~14pt (이전 단락과의 구분)

## 9. 기준선 그리드 (Baseline Grid) 활용

기준선 그리드는 여러 컬럼이나 프레임의 텍스트가 **같은 수평선에 정렬**되도록 돕는 보이지 않는 수평 격자입니다.

`기준선 그리드가 OFF일 때:          기준선 그리드가 ON일 때:
┌──────┬──────┐                   ┌──────┬──────┐
│텍스트│텍스트│                   │텍스트│텍스트│──── 기준선
│내용이│내용이│                   │내용이│내용이│
│  여기│여기서│  ← 줄이 어긋남    │  여기│여기서│──── 기준선 (정렬)
│  끝남│ 끝남 │                   │  끝남│ 끝남 │
└──────┴──────┘                   └──────┴──────┘`

### 9.1 기준선 그리드 설정

**문서 전체 기준선 그리드 설정:**

- `View` → `Baseline Grid Settings`
- **Grid Spacing:** 줄 간격과 동일한 값 또는 그 배수로 설정
  - 예: 본문 줄 간격 15pt → Grid Spacing: 15pt
- **Colour:** 그리드 선 색상 (하늘색 권장)
- **Start Position:** 첫 번째 그리드 선의 시작 위치

**그리드 표시/숨기기:**

- `View` → `Show Baseline Grid`
- 단축키: `⌘ Cmd + Shift + '` (Mac) / `Ctrl + Shift + '` (Win)

### 9.2 텍스트 프레임에 기준선 그리드 연동

1. 기준선 그리드를 사용할 텍스트 프레임 선택
2. Text Frame 패널 → **`Use Baseline Grid`** 체크
3. 해당 프레임의 텍스트가 자동으로 그리드에 스냅됨

또는 단락 스타일에서 설정:

- Edit Text Style 대화 상자 → **Paragraph** 섹션 → **Align to Baseline Grid** 체크

> 💡 **언제 기준선 그리드를 써야 하나요?**
> 2단 이상의 컬럼 레이아웃에서 양쪽 컬럼의 텍스트 줄이 나란히 정렬되어야 할 때 특히 유용합니다. 단면 주보보다는 4면 이상의 소책자에서 효과가 두드러집니다.

## 10. 타이포그래피 고급 기능

### 10.1 OpenType 기능 — Typography 패널

OpenType 폰트가 지원하는 고급 타이포그래피 기능을 적용할 수 있습니다.

**패널 열기:**

- `Text` → `Show Typography`

**주요 기능:**

| 기능 | 설명 | 사용 예 |
| --- | --- | --- |
| **Ligatures (합자)** | 두 글자가 결합된 특수 글리프 적용 | fi, fl 조합 등 |
| **Alternates (대체 글리프)** | 같은 글자의 다른 디자인 형태 선택 | 장식체 스크립트 폰트 |
| **Stylistic Sets** | 폰트별 스타일 세트 적용 | 폰트마다 다름 |
| **Figure Style (숫자 스타일)** | Lining(기준선 정렬) vs Old Style(소문자형) 숫자 | 본문 내 날짜 표기 |
| **Capitals (대문자 변형)** | Small Caps, All Caps, 등 | 제목의 단어 강조 |
| **Fractions (분수)** | 1/2, 3/4 등을 분수 글리프로 변환 | 악보 박자 표기 등 |

> 📌 **OpenType 기능은 해당 폰트가 해당 기능을 지원해야 사용 가능합니다.** 지원하지 않는 기능은 Typography 패널에서 회색으로 비활성화됩니다.
>
> 한국어 폰트(Noto Serif KR 등)는 한글 전용 글리프 처리에 집중하여 Ligatures 등 일부 서양 타이포그래피 기능은 지원하지 않습니다.

### 10.2 Drop Cap (드롭 캡)

단락의 첫 글자를 크게 키워 장식적으로 표시하는 기법입니다.

**설정 방법:**

1. 드롭 캡을 적용할 단락에 커서 위치
2. Paragraph 패널 → **Drop Cap** 섹션 확장
3. **Lines:** 드롭 캡 높이 (몇 줄 높이인지, 예: 3)
4. **Glyph Count:** 드롭 캡으로 만들 글자 수 (보통 1)
5. 드롭 캡에 문자 스타일을 별도 적용하여 색상이나 글꼴 변경 가능

`[드롭 캡 적용 예시]

┌──────────────────────────────┐
│ ┌───┐                        │
│ │ 예 │배를 드리며 우리는 한   │
│ │   │자리에 모였습니다. 오늘  │
│ └───┘도 주님의 은혜로…        │
└──────────────────────────────┘`

### 10.3 경로를 따르는 텍스트 (Text on a Path)

원형, 곡선 등 경로를 따라 텍스트를 배치할 수 있습니다.

**방법:**

1. Vector Studio에서 곡선 또는 원형 경로 생성
2. Layout Studio로 돌아와 Artistic Text Tool(`A`) 선택
3. 경로 위에 커서를 올려 경로 아이콘이 표시되면 클릭
4. 경로를 따라 텍스트 입력

**사용 예:** 원형 교회 로고 텍스트, 장식적 섹션 구분 곡선 텍스트

### 10.4 스펠 체크 (Spell Check)

- `Edit` → `Spelling` → `Check Spelling`
- 단축키: `⌘ Cmd + ;` (Mac) / `F7` (Win)
- 언어 설정이 올바르게 되어 있어야 합니다 (섹션 7.1 참조)

> 📌 **한국어 맞춤법:** Affinity의 한국어 맞춤법 검사 기능은 제한적입니다. 중요한 한국어 텍스트는 별도 맞춤법 검사 도구(한국어 워드 프로세서 등)로 검토 후 붙여넣기를 권장합니다.

## 11. 찾기 및 바꾸기 (Find & Replace)

매주 반복 작업인 주보 제작에서 날짜, 이름, 성경 구절 번호 등을 일괄 변경할 때 매우 유용합니다.

### 11.1 Find & Replace 열기

- **Mac:** `⌘ Cmd + H`
- **Windows:** `Ctrl + H`
- 또는 `Edit` → `Find and Replace`

### 11.2 주요 기능

| 기능 | 설명 |
| --- | --- |
| **Find:** | 찾을 텍스트 입력 |
| **Replace with:** | 바꿀 텍스트 입력 |
| **Replace** | 현재 발견된 항목 하나만 교체 |
| **Replace All** | 문서 전체에서 일괄 교체 |
| **Match Case** | 대소문자 구분 |
| **Whole Word** | 단어 전체 일치만 검색 |
| **Regular Expressions** | 정규식 패턴으로 검색 (고급) |

### 11.3 실전 활용 예시

**매주 날짜 업데이트:**

- Find: `2025년 5월 11일` → Replace: `2025년 5월 18일`

**설교자 이름 변경:**

- Find: `김목사` → Replace: `이목사`

**성경 번역 표기 통일:**

- Find: `NIV` → Replace: `ESV`

## 13. 용어 해설

| 용어 | 설명 |
| --- | --- |
| **Frame Text Tool** | 고정 크기 박스 안에 텍스트를 배치하는 도구. 단축키 `T`. Layout의 핵심 도구. |
| **Artistic Text Tool** | 자유롭게 늘어나는 텍스트 오브젝트를 만드는 도구. 단축키 `A`. 제목, 장식 텍스트에 사용. |
| **Text Frame** | Frame Text Tool로 생성한 텍스트 박스. 오버플로우, 컬럼, 래핑 등 제어 가능. |
| **Overflow (오버플로우)** | 텍스트 내용이 프레임보다 많아 일부가 보이지 않는 상태. 빨간 삼각형으로 표시. |
| **Text Flow (텍스트 흐름)** | 여러 프레임을 연결하여 텍스트가 이어지도록 하는 기능. |
| **AutoFlow** | 오버플로우 텍스트를 자동으로 새 페이지 프레임에 배분하는 기능. |
| **Text Wrapping (텍스트 래핑)** | 이미지나 도형 주변으로 텍스트가 흐르도록 설정하는 기능. |
| **Insets (내부 여백)** | 텍스트 프레임 테두리와 텍스트 내용 사이의 내부 여백. |
| **Leading (줄 간격)** | 줄과 줄 기준선 사이의 수직 거리. pt 또는 % 단위. |
| **Tracking (트래킹)** | 선택된 텍스트 전체의 글자 간격을 균일하게 넓히거나 좁히는 설정. |
| **Kerning (커닝)** | 특정 두 글자 사이의 간격만 미세하게 조정하는 설정. |
| **Baseline Grid (기준선 그리드)** | 텍스트 줄이 일정한 수평선에 정렬되도록 돕는 보이지 않는 수평 격자. |
| **Drop Cap (드롭 캡)** | 단락 첫 글자를 크게 키워 장식적으로 표시하는 타이포그래피 기법. |
| **Text on a Path** | 곡선이나 원형 경로를 따라 텍스트를 배치하는 기능. |
| **Ligature (합자)** | 두 글자가 하나의 글리프로 결합된 형태. OpenType 기능의 일부. |
| **Space Before/After** | 단락 시작 전/끝 후의 수직 공백. Enter로 줄 띄우는 것보다 정확한 방법. |
| **Justify (양쪽 정렬)** | 텍스트를 좌우 여백에 딱 맞게 정렬. 글자 간격 자동 조정. |
| **Gutter (거터)** | 다단 레이아웃에서 단과 단 사이의 공백. |
| **Find & Replace** | 문서 내 특정 텍스트를 검색하고 일괄 교체하는 기능. |

## 14. FAQ

**Q1. 텍스트 프레임과 Artistic Text의 차이를 언제 실감하나요?**

A. 표지의 교회 이름처럼 크기와 위치를 직접 조작하고 변형도 주고 싶은 텍스트라면 Artistic Text가 편합니다. 반면 본문 기사처럼 프레임 경계 안에 텍스트가 머물고, 경우에 따라 다음 페이지 프레임으로 이어져야 한다면 Frame Text가 필수입니다. 주보 작업의 95%는 Frame Text Tool을 사용합니다.

---

**Q2. 오버플로우 텍스트가 발생했는데 삭제하지 않고 관리하는 방법이 있나요?**

A. 오버플로우 텍스트는 숨겨진 상태일 뿐 삭제되지 않습니다. 세 가지 방법으로 처리하세요:

1. 프레임 크기를 늘려 오버플로우 텍스트가 보이도록 함
2. 다음 프레임에 연결하여 흐름을 이어줌
3. 텍스트 내용을 줄여 프레임 안에 맞게 편집

---

**Q3. 한국어 텍스트와 영어 텍스트를 같은 프레임에 넣었는데 자간이 이상합니다.**

A. 한영 혼용 텍스트에서 자간 차이가 발생하는 원인 두 가지:

1. **폰트 불일치:** 한국어 폰트와 영어 폰트의 글자 너비 기준이 다릅니다. 단락 스타일에 `Noto Serif KR`을 지정하면 영문에도 이 폰트가 적용됩니다. 영문만 별도 폰트로 바꾸려면 영문 부분을 선택하고 문자 스타일 또는 직접 폰트 변경.
2. **Tracking 값 확인:** 한국어 본문에서 Tracking이 과도하게 설정되어 있는지 확인. 기본값(0)에 가깝게 유지.

---

**Q4. 양쪽 정렬(Justify)을 적용했더니 공백이 너무 벌어지는 단어가 생깁니다.**

A. 이는 `Justify` 정렬의 단점으로, 짧은 단어가 많은 줄에서 발생합니다. 해결 방법:

1. **Justify Last Line Left** 사용 (마지막 줄의 간격 왜곡 방지)
2. 해당 단락의 **Hyphenation(하이픈 처리)** 활성화 → 단어를 줄 나눔하여 공백 분산
  - Paragraph 패널 → Hyphenation 체크
3. 특히 문제가 되는 단어에서 강제 줄바꿈: `Shift + Enter` (소프트 리턴)

---

**Q5. 기준선 그리드를 켰더니 텍스트 줄 간격이 이상하게 커졌습니다.**

A. 기준선 그리드 간격(Grid Spacing)이 단락 스타일의 줄 간격(Leading)과 다를 때 발생합니다. 해결 방법:

1. `View` → `Baseline Grid Settings`에서 Grid Spacing 값을 단락 스타일의 Leading 값과 맞춤
2. 또는 Text Frame 패널 → `Start Position` 조정으로 텍스트 시작 위치 맞추기
3. 그리드가 불필요한 프레임이라면 해당 프레임의 `Use Baseline Grid` 체크 해제

---

**Q6. Find & Replace로 날짜를 바꿨는데 일부는 바뀌지 않았습니다.**

A. 다음 원인을 확인하세요:

1. **마스터 페이지의 텍스트:** Find & Replace는 기본적으로 일반 페이지 텍스트만 검색합니다. 마스터 페이지 텍스트는 마스터 페이지 편집 모드에서 별도로 변경해야 합니다.
2. **검색 옵션:** `Match Case` 체크로 대소문자가 다른 텍스트 미검색될 수 있음
3. **텍스트 프레임 잠금:** Layers 패널에서 해당 프레임이 잠겨 있으면 교체 안 됨

## 15. Tips: 실수를 줄이고 더 잘 사용하는 법

### 💡 Tip 1: 텍스트 프레임은 먼저 레이아웃을 잡고 내용을 채우자

반접기 주보 작업 시, 빈 텍스트 프레임을 먼저 모두 배치하고 크기를 정한 다음 텍스트를 입력하면 레이아웃을 유지하기 쉽습니다. 내용을 먼저 입력하면 텍스트 양에 따라 레이아웃이 계속 흔들립니다.

**추천 순서:**

`1. 빈 프레임 배치 → 크기·위치 확정
2. 필요 시 프레임 연결
3. 텍스트 입력 (Word에서 복사 붙여넣기 포함)
4. 스타일 적용
5. 세부 조정`

---

### 💡 Tip 2: `Shift + Enter`로 단락 구분 없이 줄을 바꾸자

`Enter` 키는 **새 단락**을 시작합니다 (Space Before/After 적용됨). 같은 단락 안에서 단순히 줄만 바꾸고 싶을 때는 `Shift + Enter`를 사용하세요. 날짜 + 장소처럼 한 단락으로 묶어야 하는 두 줄에 유용합니다.

`Enter:
"2025년 5월 18일 주일 오전 11시"     ← 단락 1 (Space After 적용)

"예배당 본관"                        ← 단락 2

Shift + Enter:
"2025년 5월 18일 주일 오전 11시      ← 같은 단락, 줄만 바꿈
예배당 본관"                           (Space After 없음)`

---

### 💡 Tip 3: 컬럼은 별도 프레임보다 Text Frame 패널에서 설정하자

2단 레이아웃이 필요할 때, 두 개의 별도 프레임을 만들고 연결하는 방법도 있지만, Text Frame 패널의 Columns 설정을 사용하면 훨씬 간단합니다. 하나의 프레임 안에서 컬럼이 자동으로 균등하게 나뉘고, 첫 번째 컬럼이 차면 자동으로 두 번째 컬럼으로 넘어갑니다.

---

### 💡 Tip 4: 텍스트 프레임에 배경색을 넣을 때 Insets를 꼭 설정하자

텍스트 프레임에 배경색을 추가하면 텍스트가 프레임 가장자리에 붙어 보일 수 있습니다. Text Frame 패널의 **Insets** (내부 여백)를 0.1~0.15"로 설정하면 텍스트와 배경 테두리 사이에 여백이 생겨 가독성이 높아집니다.

---

### 💡 Tip 5: 매주 주보는 이전 파일을 복제해서 시작하자

매주 새 파일을 만드는 것보다, 지난 주 완성 파일을 복제(`File → Duplicate`)하거나 사본으로 저장(`File → Save As`)하여 텍스트만 교체하는 방식이 훨씬 효율적입니다. 레이아웃, 스타일, 마스터 페이지가 그대로 유지되므로 변경 사항만 처리하면 됩니다.

---

### 💡 Tip 6: 텍스트 입력 전 언어 확인을 습관화하자

한국어 텍스트 프레임에 영어 텍스트를 그대로 붙여넣으면 언어 설정이 섞일 수 있습니다. 단락 스타일을 올바르게 적용한 다음 텍스트를 붙여넣으면 해당 스타일의 언어 설정이 유지됩니다.
