# 제8장: 생산성 최적화와 지속가능한 실천 – Second Brain을 삶의 일부로 만들기

당신은 Second Brain을 구축했습니다. 노트는 PARA로 정리되어 있고, CODE 시스템으로 정보를 처리하며, 도구도 선택했습니다. 하지만 3개월 후, 많은 사람들이 똑같은 함정에 빠집니다.

시스템이 부담이 되기 시작합니다. 노트는 쌓이지만 리뷰하지 않고, 프로젝트는 시작하지만 끝내지 못하며, 수집은 계속하지만 창작은 줄어듭니다. 결국 "이 시스템이 나를 더 생산적으로 만드는가, 아니면 더 바쁘게만 만드는가?" 자문하게 됩니다.

**진실은 이것입니다: Second Brain은 당신을 위해 일해야지, 당신이 시스템을 위해 일해서는 안 됩니다.**

이번 챕터는 Second Brain을 지속 가능한 삶의 일부로 만드는 방법을 다룹니다. GTD와의 통합, 에너지 관리, 습관 설계, 그리고 가장 중요한 것 - 번아웃 없이 시스템을 유지하는 방법까지. 이것은 단순한 생산성 해킹이 아니라, 장기적으로 창의적이고 균형잡힌 삶을 위한 전략입니다.

---

## 8.1 GTD와 세컨드 브레인의 통합

### 8.1.1 두 거장의 만남: David Allen과 Tiago Forte

GTD(Getting Things Done)와 Second Brain은 서로 경쟁하는 시스템이 아닙니다. 오히려 **완벽한 시너지를 만드는 동반자**입니다.

**GTD가 강한 부분:**

- 작업과 약속 관리
- 스트레스 없는 생산성
- 명확한 다음 행동
- 체계적인 리뷰 프로세스

**Second Brain이 강한 부분:**

- 지식과 아이디어 관리
- 창의적 연결과 통찰
- 장기적 학습과 성장
- 재사용 가능한 자산 구축

**철학적 일치점:**

> "당신의 마음은 아이디어를 갖기 위한 것이지, 붙잡아두기 위한 것이 아니다."
> — David Allen

이 원칙은 두 시스템의 핵심입니다. GTD는 작업을, Second Brain은 지식을 외부화합니다.

### 8.1.2 통합 아키텍처

**PARA와 GTD의 매핑:**

```plain text
GTD 구조                 →    PARA 구조
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Projects List           →    Projects 폴더
Areas of Focus         →    Areas 폴더
Reference Materials    →    Resources 폴더
Completed/Inactive     →    Archives 폴더

```

**구체적 구현:**

```markdown
📁 Second Brain
  📁 00_Inbox (GTD 수집함)
    📄 빠른 캡처
    📄 이메일에서 온 것
    📄 회의 메모

  📁 01_Projects (GTD 프로젝트)
    📂 웹사이트 리뉴얼 [2025-02-28]
      📄 프로젝트 계획
      📄 다음 행동: @computer
      📄 대기 중: @waiting_for
      📄 관련 자료

  📁 02_Areas (GTD 책임 영역)
    📂 건강 관리
      📄 운동 루틴
      📄 의료 기록
      📄 정기 체크업 일정

  📁 03_Resources (GTD 참고 자료)
  📁 04_Archives (GTD 완료/보류)

```

### 8.1.3 수집함(Inbox) 관리

**통합 수집 시스템:**

```plain text
모든 입력 → 단일 Inbox → 처리 → 적절한 위치

입력 소스:
- 이메일
- 메모 앱
- 음성 메모
- 웹 클리핑
- 물리적 서류

```

**2분 규칙 + Progressive Summarization:**

```markdown
수집 항목 도착
  ↓
2분 안에 끝나는가?
  ├─ Yes → 즉시 실행
  └─ No → Second Brain으로
           ↓
         Project 관련? → Projects 폴더 + 다음 행동 정의
         Area 관련? → Areas 폴더 + 기준 업데이트
         참고 자료? → Resources + Layer 1 강조
         필요 없음? → 삭제

```

**일일 Inbox Zero 프로세스 (15분):**

```markdown
## 📥 Inbox 처리 체크리스트

### 1. 수집 (2분)
- [ ] 모든 물리적 메모 디지털화
- [ ] 이메일 플래그 항목 이동
- [ ] 음성 메모 전사

### 2. 명확화 (5분)
각 항목에 대해:
- 이것은 무엇인가?
- 실행 가능한가?
- 다음 행동은?

### 3. 정리 (5분)
- [ ] 2분 규칙 적용
- [ ] PARA 폴더로 이동
- [ ] 다음 행동 태그 추가

### 4. 검토 (3분)
- [ ] Inbox 비움 확인
- [ ] 오늘의 MIT 확인

```

### 8.1.4 다음 행동 정의

**GTD의 "다음 행동"을 Second Brain에 통합:**

```markdown
❌ 모호한 작업:
"웹사이트 개선"

✅ 명확한 다음 행동:
"@phone: 디자이너에게 전화 - 홈페이지 목업 피드백 (15분)"

```

**컨텍스트 태그 시스템:**

```plain text
@computer - 컴퓨터 필요
@phone - 전화 가능할 때
@home - 집에서만
@office - 사무실에서만
@errands - 외출 시
@waiting - 대기 중
@energy_high - 에너지 높을 때
@energy_low - 피곤할 때 가능

```

**실전 예시:**

```markdown
# 프로젝트: 블로그 리뉴얼

## 🎯 다음 행동
- [ ] @computer: 경쟁 블로그 3개 분석 (30분) #energy_high
- [ ] @phone: 호스팅 업체 문의 - 플랜 업그레이드 (10분)
- [ ] @waiting: 디자이너 목업 받기 (due: 11/25)

## 📋 향후 행동 (다음 행동 완료 후)
- [ ] 콘텐츠 마이그레이션 계획
- [ ] SEO 체크리스트 작성

```

## 8.2 시간과 에너지 관리

### 8.2.1 Time Blocking과 Second Brain

**Time Blocking은 캘린더를 캔버스로 사용하는 것입니다.**

```plain text
❌ 할 일 목록만 있는 경우:
- 10개 작업
- 우선순위 불명확
- 시간 예측 없음
- 결과: 3개만 완료, 죄책감

✅ Time Blocking:
09:00-10:30 | Deep Work: 프로젝트 A 작성
10:30-11:00 | Email 처리 + Inbox 정리
11:00-12:00 | 회의: 팀 스탠드업
14:00-16:00 | Deep Work: 코드 리뷰
16:00-16:30 | Second Brain 리뷰

```

**Second Brain과 Time Block 연결:**

```markdown
## 🗓️ 월요일 Time Blocks

### 09:00-10:30: Deep Work Block
- 프로젝트: [[Q4 마케팅 전략]]
- 목표: 섹션 3 완성
- 자료: [[경쟁사 분석]], [[고객 페르소나]]
- 산출물: 초안 v2

### 14:00-16:00: Creative Block
- 작업: 블로그 포스트 작성
- 노트: [[콘텐츠 아이디어]], [[SEO 키워드]]
- 템플릿: [[블로그 포스트 템플릿]]

```

### 8.2.2 에너지 곡선 활용

**당신의 에너지는 하루 종일 일정하지 않습니다.**

**에너지 매핑 연습 (1주일):**

```markdown
## 🔋 나의 에너지 패턴

### 고에너지 시간대 (9-11am, 2-4pm)
최적 작업:
- 복잡한 문제 해결
- 창의적 작업
- 중요한 의사결정
- Deep Work

### 중간 에너지 (11am-1pm, 4-6pm)
최적 작업:
- 회의와 협업
- 이메일 처리
- 루틴 작업
- 리뷰와 계획

### 저에너지 (1-2pm, 6pm 이후)
최적 작업:
- 단순 정리
- Second Brain 정리
- 읽기와 학습
- Inbox 처리

```

**에너지 기반 작업 할당:**

```plain text
아침 (High):
→ Projects 폴더의 가장 중요한 작업
→ Progressive Summarization Layer 3-4

점심 후 (Low):
→ Resources 폴더 정리
→ 이메일 처리, Inbox 비우기

오후 (High):
→ 창의적 작업, 글쓰기
→ 복잡한 연결 만들기

저녁 (Low):
→ 내일 계획
→ 간단한 노트 정리

```

### 8.2.3 Deep Work와 Shallow Work 구분

**Cal Newport의 Deep Work를 Second Brain과 통합:**

```markdown
## 🎯 Deep Work Sessions

### 정의
"인지적으로 까다로운 활동에 방해받지 않고 집중하여
새로운 가치를 창출하고 기술을 향상시키는 작업"

### Deep Work 준비 (5분)
1. 목표 명확히: "이 세션에서 달성할 것"
2. Second Brain에서 관련 노트 모으기
3. 필요한 Resources 열기
4. 방해 요소 제거 (폰, 알림, 탭)

### Deep Work 실행 (90분)
- Timer 설정
- 단일 프로젝트 집중
- 중간 아이디어 → 빠른 메모
- 절대 멀티태스킹 금지

### Deep Work 마무리 (5분)
- 산출물 저장
- 다음 세션을 위한 메모
- 진행 상황 기록

```

**Shallow Work 배치 처리:**

```markdown
## 📋 Shallow Work Batches

### Email Batch (30분, 하루 2회)
- 09:30-10:00
- 16:30-17:00
- 템플릿 사용으로 속도 향상

### Admin Batch (30분, 주 3회)
- 월/수/금 17:00-17:30
- 경비 처리
- 일정 조정
- 서류 작업

### Second Brain Maintenance (15분, 매일)
- 17:30-17:45
- Inbox 처리
- 빠른 링크 추가
- 내일 준비

```

## 8.3 습관 형성과 루틴 구축

### 8.3.1 모닝 루틴: Second Brain 시동걸기

**효과적인 아침 루틴 템플릿:**

```markdown
## ☀️ Morning Routine (30분)

### 6:30-6:35: Mind Dump (5분)
- [ ] 침대 옆 노트에 떠오르는 생각 모두 적기
- [ ] 꿈, 아이디어, 걱정 모두 OK
- [ ] 판단하지 말고 그냥 쓰기

### 6:35-6:45: Review & Plan (10분)
- [ ] 어제 완료한 것 체크
- [ ] 오늘의 MIT 3개 선정
- [ ] Time Block 확인/조정

### 6:45-6:55: Prime the Brain (10분)
- [ ] 현재 프로젝트 노트 스캔
- [ ] 관련 Resources 빠르게 검토
- [ ] 오늘 필요한 정보 식별

### 6:55-7:00: Set Intention (5분)
- [ ] 오늘의 한 문장 목표
- [ ] 에너지 레벨 체크
- [ ] Deep Work 시간 확정

```

**실제 사례: 소프트웨어 엔지니어 김민수님**

```plain text
"매일 아침 15분 Second Brain 리뷰로 시작합니다.
어제 캡처한 것 정리 → 오늘 프로젝트 확인 →
관련 노트 연결. 이것만으로 오전 생산성이 2배
올랐습니다. 특히 코드 리뷰할 때 관련 문서를
미리 준비해두니 시간이 절반으로 줄었어요."

```

```markdown
### 17:15-17:20: Tomorrow Prep (5분)
- [ ] 내일 MIT 확인

### 완료 의식

```

### 8.3.3 습관 스태킹 with Second Brain

**Habit Stacking 공식:**
"[현재 습관] 후에, [새로운 Second Brain 습관]을 한다"

**실전 예시:**

```markdown
## 🔗 My Habit Stacks

### 아침 커피 → 노트 리뷰
"커피를 마시면서, 어제 캡처한 노트 3개를 리뷰한다"
- Trigger: 커피 첫 모금
- Action: Inbox 열기
- Reward: 하나씩 처리할 때마다 체크

### 점심 후 → 빠른 캡처
"점심 먹고 자리로 돌아오면, 오전 회의 메모를 정리한다"
- Trigger: 의자에 앉기
- Action: 회의록 템플릿 열기
- Time: 최대 10분

### 퇴근 전 → Inbox Zero
"컴퓨터 끄기 전에, Inbox를 비운다"
- Trigger: 캘린더 알림 (17:00)
- Action: 모든 항목 처리
- Reward: 깨끗한 시작

### 주말 카페 → 주간 리뷰
"토요일 카페에서, 주간 리뷰를 한다"
- Trigger: 카페 도착
- Action: 리뷰 템플릿 시작
- Reward: 디저트 주문

```

## 8.4 정기 리뷰 시스템

### 8.4.1 일일 검토 (5분)

```markdown
## 📅 Daily Review Template

**시간**: 17:00-17:05 (매일)

### ✅ Quick Wins
- [ ] 오늘 완료한 작업 3개
- [ ] 캡처한 인사이트 1개
- [ ] 만든 연결 1개

### 📝 Quick Process
- [ ] Inbox 항목 수: [  ] → [  ]
- [ ] 긴급 처리 필요: Yes/No

### 🎯 Tomorrow's Focus
- MIT #1:
- MIT #2:
- MIT #3:

시간: _____분 (목표: 5분 이내)

```

### 8.4.2 주간 검토: GTD + Second Brain

**통합 주간 리뷰 프로세스 (45분):**

```markdown
## 🗓️ Weekly Review [날짜]

### Phase 1: GET CLEAR (15분)

#### 수집 (5분)
- [ ] 모든 Inbox 비우기
- [ ] 느슨한 종이들 처리
- [ ] 음성 메모 전사

#### 처리 (10분)
- [ ] 이메일 Inbox Zero
- [ ] Second Brain Inbox Zero
- [ ] 메시지 앱 확인

### Phase 2: GET CURRENT (20분)

#### 프로젝트 리뷰 (10분)
- [ ] 각 Project 폴더 열기
- [ ] 다음 행동 업데이트
- [ ] 완료 → Archives 이동
- [ ] 새 프로젝트 추가

#### Areas 체크 (5분)
- [ ] 각 Area 기준 확인
- [ ] 주의 필요 영역 식별
- [ ] 필요시 프로젝트 생성

#### 캘린더 리뷰 (5분)
- [ ] 지난 주 일정 검토
- [ ] 다음 2주 확인
- [ ] 준비 필요 사항

### Phase 3: GET CREATIVE (10분)

#### Someday/Maybe 검토
- [ ] 시작할 준비된 것?
- [ ] 새로운 아이디어 추가

#### 큰 그림 질문
- 현재 가장 중요한 것은?
- 무엇을 미루고 있는가?
- 시스템 개선 필요?

### 📊 주간 메트릭
- 완료 프로젝트: [  ]
- 생성한 노트: [  ]
- 만든 연결: [  ]
- Deep Work 시간: [  ]h

```

### 8.4.3 월간 심층 리뷰


## 8.5 번아웃 방지와 지속가능성

### 8.5.1 PKM 번아웃의 징후

**경고 신호를 무시하지 마세요:**

```markdown
## 🚨 번아웃 체크리스트

### 초기 징후
- [ ] 노트 정리에 실제 작업보다 더 많은 시간 소비
- [ ] "완벽한 시스템"을 찾아 도구 계속 변경
- [ ] 캡처는 많지만 창작은 없음
- [ ] 시스템 유지가 부담스러움

### 중기 징후
- [ ] 노트 리뷰 건너뛰기 시작
- [ ] Inbox가 계속 쌓임
- [ ] 프로젝트 시작만 하고 완료 못함
- [ ] 시스템에 대한 죄책감

### 심각한 징후
- [ ] Second Brain 완전히 방치
- [ ] "차라리 없는 게 낫다" 생각
- [ ] 정보 관리에 대한 불안감
- [ ] 창의성과 생산성 급감

```

### 8.5.2 완벽주의 극복

**"완벽한 시스템"의 함정:**

```plain text
❌ 완벽주의자의 Second Brain:
- 15단계 태그 체계
- 색상 코딩된 7레벨 폴더
- 모든 노트 매일 리뷰
- 100% 연결된 노트
- 결과: 3개월 후 포기

✅ 실용주의자의 Second Brain:
- 4개 PARA 폴더
- 5-10개 핵심 태그
- 주간 리뷰만
- 중요한 것만 연결
- 결과: 3년째 사용 중

```

**80/20 원칙 적용:**

```markdown
## 🎯 나의 20% (핵심만 남기기)

### 가장 많이 쓰는 기능 (Keep)
1. 빠른 캡처
2. 검색
3. 프로젝트 노트
4. 주간 리뷰

### 거의 안 쓰는 기능 (Remove)
1. 복잡한 태그 체계
2. 상세한 메타데이터
3. 일일 리뷰
4. 완벽한 포맷팅

### 시간 할당
- 캡처: 10%
- 정리: 20%
- 창작: 70% ← 이것이 목표!

```

### 8.5.3 점진적 개선 마인드셋

**1% 개선의 복리 효과:**

```plain text
매일 1% 개선:
1.01^365 = 37.78

매주 작은 개선 예시:
Week 1: 캡처 단축키 설정 (30초 절약)
Week 2: 템플릿 1개 추가 (5분 절약)
Week 3: 태그 단순화 (정신적 부담 감소)
Week 4: 주간 리뷰 루틴화

1년 후: 완전히 다른 수준의 효율성

```

**개선 실험 로그:**

```markdown
## 🧪 시스템 실험 로그

### 실험 1: 아침 캡처 시간 변경
- 가설: 침대 옆 노트가 폰보다 효과적
- 기간: 2주
- 결과: ✅ 성공 - 아이디어 캡처 3배 증가
- 채택: Yes

### 실험 2: 일일 리뷰 추가
- 가설: 매일 5분 리뷰가 주간 리뷰 부담 감소
- 기간: 1주
- 결과: ❌ 실패 - 너무 빈번, 부담됨
- 채택: No (주 3회로 조정)

### 실험 3: 음성 메모 전환
- 가설: 운전 중 음성 메모가 효과적
- 기간: 1개월
- 결과: ✅ 성공 - 아이디어 손실 90% 감소
- 채택: Yes

```

### 8.5.4 시스템 단순화 전략

**복잡성 감사 (Complexity Audit):**

```markdown
## 🔍 시스템 복잡성 감사

### 현재 복잡성 측정
- 폴더 깊이: 최대 ___단계
- 태그 수: ___개
- 사용 중인 도구: ___개
- 일일 프로세스: ___개
- 주간 프로세스: ___개

### 단순화 목표
- 폴더 깊이: 최대 2단계
- 태그: 10개 이하
- 핵심 도구: 2-3개
- 일일 프로세스: 3개 이하

### 제거 후보
1. 3개월간 사용 안한 태그
2. 비어있는 폴더
3. 중복 프로세스
4. 가치 없는 루틴

```

**미니멀 Second Brain 템플릿:**

```plain text
📁 Simple Second Brain
  📁 Now (현재 활성 프로젝트만, 최대 5개)
  📁 Next (다음 할 것들)
  📁 Notes (모든 참고 자료)
  📁 Done (완료된 것들)

태그: 없음 (검색만 사용)
리뷰: 주 1회만
도구: 노트 앱 1개 + 클라우드 저장소

```

## 8.6 우선순위와 집중력

### 8.6.1 아이젠하워 매트릭스 with Second Brain

```markdown
## ⚡ Eisenhower Matrix + PARA

### Q1: 긴급 + 중요 (Do First)
→ Projects 폴더의 "🔥" 태그
- 마감 임박 프로젝트
- 위기 상황
- 즉시 처리

### Q2: 중요 + 긴급하지 않음 (Schedule)
→ Projects + Areas의 대부분
- 장기 프로젝트
- 능력 개발
- 관계 구축
- **여기에 시간의 60% 투자**

### Q3: 긴급 + 중요하지 않음 (Delegate)
→ 자동화 또는 위임
- 반복적 작업 → 템플릿화
- 정보 요청 → FAQ 문서
- 미팅 → 녹화 + 요약

### Q4: 긴급하지도 중요하지도 않음 (Delete)
→ 하지 않기
- 과도한 노트 정리
- 완벽한 태그 시스템
- "나중에 볼" 콘텐츠

```

### 8.6.2 MIT (Most Important Tasks) 시스템

**일일 MIT 선정:**

```markdown
## 🎯 Today's MITs [날짜]

### MIT #1: [프로젝트 핵심 작업]
- Project: [[프로젝트명]]
- Time Block: 09:00-11:00
- Resources: [[관련 노트]]
- Success Metric: [완료 기준]

### MIT #2: [중요한 커뮤니케이션]
- Context: [[배경 정보]]
- Time Block: 14:00-15:00
- Outcome: [원하는 결과]

### MIT #3: [학습/성장]
- Area: [[관련 영역]]
- Time Block: 16:00-17:00
- Note: [[학습 노트]]

### Buffer Tasks (MIT 완료 후)
- [ ] 이메일 처리
- [ ] Inbox 정리
- [ ] 내일 준비

```

**주간 MIT 계획:**

```plain text
월: 프로젝트 A 핵심 작업
화: 클라이언트 미팅 준비
수: 콘텐츠 작성
목: 프로젝트 B 마일스톤
금: 주간 리뷰 + 다음 주 계획

원칙:
- 하루 최대 3개 MIT
- 1개 이상은 Q2 영역에서
- 완료 못하면 다음 날 최우선

```

### 8.7.1 팀 단위 Second Brain

```plain text
  📁 Projects (공유 프로젝트)
    📂 Q4 제품 런칭
      📄 프로젝트 브리프
      📄 태스크 분배
      📄 진행 상황

  📁 Areas (팀 책임 영역)
    📂 고객 지원
    📂 제품 개발
    📂 마케팅

  📁 Resources (공유 지식)
    📂 프로세스 문서
    📂 템플릿
    📂 가이드라인

  📁 Archive (과거 프로젝트)

```

### 8.7.2 지식 공유 문화 구축

**공유 규칙과 에티켓:**

```markdown
## 📢 팀 지식 공유 가이드라인

### 무엇을 공유할 것인가
✅ 공유 대상:
- 프로젝트 학습과 회고
- 유용한 템플릿과 프로세스
- 연구 결과와 인사이트
- 실수와 교훈

❌ 공유 제외:
- 개인적 메모
- 미완성 아이디어 (명시하지 않는 한)
- 민감한 정보

### 어떻게 공유할 것인가
1. 제목: 명확하고 검색 가능하게
2. 요약: 3줄 이내 핵심
3. 본문: 구조화된 정보
4. 태그: #team-learning #template #process
5. 멘션: 관련자 @태그

### 언제 공유할 것인가
- 프로젝트 마일스톤 완료 시
- 중요한 학습 발생 시
- 주간 팀 미팅에서
- 월간 지식 공유 세션

```

### 8.7.3 개인 vs 조직 지식 균형

**경계 설정 프레임워크:**

```markdown
## 🔐 Knowledge Boundaries

### 100% 개인 (Private)
- 개인 일기와 성찰
- 민감한 피드백 메모
- 커리어 계획
- 개인 재무

### 선택적 공유 (Selective)
- 프로젝트 아이디어
- 실험 결과
- 학습 노트
- 피드백과 개선 제안

### 100% 공유 (Public)
- 프로세스 문서
- 템플릿
- 베스트 프랙티스
- 프로젝트 회고

### 공유 결정 트리
새 지식 생성
  ↓
팀에 도움되는가?
  ├─ No → 개인 Second Brain
  └─ Yes → 민감한가?
           ├─ Yes → 선택적 공유
           └─ No → 팀 Second Brain

```

## 8.8 다음 장을 향하여

이번 장에서 우리는 Second Brain을 단순한 정보 저장소가 아닌, **지속 가능한 생산성 시스템**으로 만드는 방법을 배웠습니다. GTD와의 통합, 에너지 관리, 습관 설계, 그리고 가장 중요한 번아웃 방지까지.

하지만 여정은 여기서 끝나지 않습니다. Second Brain은 도구와 방법론의 모음이 아니라, **평생 함께할 성장의 동반자**입니다.

부록에서는 즉시 사용할 수 있는 체크리스트, 템플릿, 그리고 추가 학습 자료를 제공합니다. 이제 당신의 Second Brain을 진정한 "제2의 뇌"로 만들 시간입니다.

---

**📚 이 챕터의 핵심 요약**

1. **GTD + Second Brain = 완벽한 시너지**
2. **에너지 관리 > 시간 관리**
3. **습관 스태킹으로 자동화**
4. **주간 리뷰는 필수, 일일 리뷰는 선택**
5. **완벽주의는 적, 80/20이 친구**
6. **단순함이 지속가능성의 열쇠**
7. **MIT로 집중력 극대화**
8. **팀 지식 공유는 선택적으로**
9. **번아웃 징후를 조기 감지하라**
10. **시스템은 당신을 위해, 당신은 시스템을 위해 일하지 않는다**

**💡 당신의 다음 액션**

1. GTD 수집함과 Second Brain Inbox 통합하기
2. 자신의 에너지 패턴 1주일 추적하기
3. 3개의 습관 스택 만들기 (기존 습관 + Second Brain)
4. 이번 주말 첫 주간 리뷰 예약하기
5. 시스템에서 가장 복잡한 부분 하나 제거하기

**Remember: 최고의 시스템은 당신이 실제로 사용하는 시스템입니다.** 🚀
