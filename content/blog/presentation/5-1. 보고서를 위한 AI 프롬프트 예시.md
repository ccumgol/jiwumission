## AI 도구 활용하기

### 프롬프트 1: 보고서 구조 잡기

```plain text
다음 행사/프로젝트의 결과 보고서 구조를 만들어주세요:

행사/프로젝트: [이름]
기간: [날짜]
규모: [인원/예산]
목적: [목적]

다음을 만들어주세요:

1. 보고서 전체 목차 (10-20페이지)
2. 각 섹션에 포함해야 할 내용 (bullet points)
3. 필요한 데이터/지표 목록
4. 증빙 자료 목록
5. 작성 시 주의사항

조건:
- 객관적, 사실 중심
- 숫자와 증거 기반
- 문제점도 포함
- 개선 사항 제시

예시:
행사/프로젝트: 교회 청년부 수련회
기간: 2025.12.20-22 (2박3일)
규모: 42명, 예산 1,500만원
목적: 영적 재충전, 교제 강화
```

### 프롬프트 2: 데이터 분석 및 시각화

```plain text
다음 데이터를 분석하고 보고서용으로 정리해주세요:

[원본 데이터 붙여넣기]

다음 작업을 해주세요:

1. 기술 통계 (평균, 중앙값, 최빈값)
2. 목표 대비 달성률 계산
3. 전년 대비 증감률 계산
4. 어떤 그래프가 효과적일지 제안
5. 주요 인사이트 3가지 추출
6. 표 형식으로 정리

예시:
[만족도 설문 원본 데이터]
응답 38건
1~5점 척도

또는

[예산 집행 데이터]
항목별 예산/집행액
```

### 프롬프트 3: 문제 상황 보고 작성

```plain text
다음 문제 상황을 보고서에 작성하려고 합니다.
객관적이고 투명하게 작성해주세요:

문제 상황: [설명]
발생 시점: [날짜/시간]
원인: [추정 원인]
대응: [취한 조치]
결과: [최종 결과]

다음 형식으로:

1. 문제 개요 (5W1H)
2. 발생 경위 (시간순)
3. 즉시 대응 조치
4. 영향 평가 (정량적)
5. 재발 방지 대책
6. 교훈/개선사항

조건:
- 책임 회피 없이
- 사실만 기술
- 감정 배제
- 건설적 개선안 제시

예시:
문제 상황: 수련회 중 청년 1명 발목 부상
발생 시점: Day 1, 20:45
원인: 계단에서 발 헛딛음
대응: 응급처치, 병원 진료
결과: 경미 염좌, 전체 일정 참여 가능
```

### 프롬프트 4: 개선 사항 도출

```plain text
다음 정보를 바탕으로 개선 사항을 제안해주세요:

[만족도 조사 결과]
[발생한 문제들]
[참석자 자유 의견]

다음 형식으로:

1. 현재 문제점 분석
   - 정량적 근거
   - 정성적 피드백

2. 개선 방안
   - 단기 (다음 회)
   - 중기 (내년)
   - 장기 (3년)

3. 각 개선안마다:
   - 구체적 실행 방법
   - 예상 효과 (정량화)
   - 소요 비용/시간
   - 담당 부서/인원

4. 우선순위 (중요도 x 시급도)

예시:
만족도 조사: 교통 항목 3.9/5.0 (최하위)
문제: 버스 입석 2명 발생
의견: "교통이 불편했다" 5건 (13%)
```

### 프롬프트 5: Executive Summary 작성

```plain text
다음 보고서 전체 내용을 1-2페이지
Executive Summary로 요약해주세요:

[보고서 전체 내용 또는 주요 포인트]

다음 내용 포함:

1. 행사/프로젝트 개요 (3-4문장)
2. 주요 성과 (3-5개 bullet points)
3. 정량적 지표 (표 형식)
4. 발생 문제 및 해결 (간략히)
5. 차기 개선 사항 (3개)
6. 종합 평가 (2문장)

조건:
- 총 200-300단어
- 숫자 중심
- 객관적 톤
- 이것만 읽고도 전체 파악 가능
```

## 체크리스트: 제출 전 최종 점검

### 7대 원칙 점검

- [ ] **5W2H**: 기본 정보가 모두 포함되어 있는가?
- [ ] **숫자**: 구체적 수치로 표현되어 있는가?
- [ ] **시간순**: 시간 순서대로 정리되어 있는가?
- [ ] **객관성**: 주관적 표현을 배제했는가?
- [ ] **증거**: 모든 주장에 근거가 있는가?
- [ ] **비교**: 의미 있는 비교가 포함되어 있는가?
- [ ] **투명성**: 문제도 솔직하게 보고했는가?

### 내용 점검 (15가지)

- [ ] 표지에 필요한 정보가 있는가?
- [ ] Executive Summary가 1-2페이지인가?
- [ ] 목적과 배경이 명확한가?
- [ ] 추진 경과가 시간순으로 정리되어 있는가?
- [ ] 주요 성과가 정량화되어 있는가?
- [ ] 목표 대비/전년 대비 비교가 있는가?
- [ ] 예산 집행이 항목별로 상세한가?
- [ ] 증빙 서류가 첨부되어 있는가?
- [ ] 발생 문제가 투명하게 보고되어 있는가?
- [ ] 문제 대응이 구체적으로 기술되어 있는가?
- [ ] SWOT 분석 또는 유사 분석이 있는가?
- [ ] 개선 사항이 구체적이고 실행 가능한가?
- [ ] 차기 계획이 제시되어 있는가?
- [ ] 첨부 자료가 충분한가?
- [ ] 연락처/문의처가 명시되어 있는가?

### 형식 점검 (10가지)

- [ ] 페이지 번호가 있는가?
- [ ] 목차가 있는가? (10페이지 이상)
- [ ] 표와 그래프가 명확한가?
- [ ] 표/그래프에 제목과 출처가 있는가?
- [ ] 숫자 단위가 일관되는가? (원, 명, %)
- [ ] 폰트와 색상이 일관되는가?
- [ ] 오타가 없는가?
- [ ] 인쇄 시 읽기 쉬운가?
- [ ] 파일명이 명확한가? (날짜, 버전 포함)
- [ ] PDF 변환이 정상적인가?
