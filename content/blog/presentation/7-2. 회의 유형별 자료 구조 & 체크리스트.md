### 회의 유형별 자료 구조

## 유형 1: 정기 진행 상황 회의 (주간/월간)

- 목적: 프로젝트/업무 진행 공유, 이슈 해결
- 권장 구조: 5~7슬라이드, 15~20분

**슬라이드 예시**

1. 요약
  - 전체 현황: 순조 (Green)
  - 완료, 진행, 지연 핵심 1줄씩
2. 주요 성과 (3개)
3. 진행 사항 (체크리스트)
4. 이슈 및 대응
5. 다음 달 계획
6. 결정 사항
7. 다음 단계 (Action Items)

---

## 유형 2: 의사결정 회의

- 목적: 중요 사안에 대한 결정
- 권장 구조: 7~10슬라이드, 30~45분

**슬라이드 예시**

1. 결정 요청 (무엇을 결정할지, 시한)
2. 배경

3~4. 옵션 비교 (표)

1. 평가 기준 (가중치)
2. 위험 분석
3. 추천 (근거)
4. 의사결정 기록 (투표 또는 합의)
5. Next Steps

---

## 유형 3: 브레인스토밍 회의

- 목적: 아이디어 도출, 창의적 해결
- 권장 구조: 5~8슬라이드, 45~60분

**슬라이드 예시**

1. 도전 과제 (Challenge)
2. 배경 데이터
3. 브레인스토밍 규칙
4. 아이디어 수집 (보드)
5. 투표/우선순위
6. 실행 가능성 분석 (2x2)
7. 실행 계획
8. Next Steps

---

## 유형 4: 임원 보고 회의

- 목적: 경영진에게 핵심 요약 보고
- 특징:
  - 시간: 10~15분
  - 슬라이드: 3~5장
  - 메시지: 명확
  - 요청: 구체

**권장 구조**

1. Executive Summary
2. 주요 성과 (3대 성과)
3. 2026 전략 (3대 방향)
4. 예산 요청 (표 + 근거)
5. 승인 요청 사항

---

## 체크리스트: 회의 전 최종 점검

- [ ] 결론 우선: 첫 슬라이드에 핵심이 있는가?
- [ ] 1슬라이드 1메시지: 각 슬라이드마다 메시지가 하나인가?
- [ ] 3의 법칙: 핵심 포인트가 3개인가?
- [ ] 데이터 시각화: 표보다 그래프를 썼는가?
- [ ] 액션 명확: 누가, 무엇을, 언제 하는지 명확한가?
- [ ] 회의 목적이 명확한가?
- [ ] 참석자를 고려했는가?
- [ ] 제한 시간 내 발표 가능한가?
- [ ] 결정이 필요한 사항이 명시되었는가?
- [ ] 배경 정보가 충분한가?
- [ ] 데이터가 정확한가?
- [ ] 시각화가 효과적인가?
- [ ] 옵션 비교가 명확한가?
- [ ] 각 옵션의 장단점이 있는가?
- [ ] 추천 의견에 근거가 있는가?
- [ ] 리스크가 식별되었는가?
- [ ] Next Steps가 명확한가?
- [ ] 담당자와 기한이 명시되었는가?
- [ ] 예상 질문에 대비했는가?
- [ ] 백업 자료가 준비되었는가?
- [ ] 슬라이드 번호가 있는가?
- [ ] 제목이 명확한가?
- [ ] 폰트 크기가 적절한가? (최소 24pt)
- [ ] 색상 대비가 충분한가?
- [ ] 애니메이션이 과하지 않은가?
- [ ] 표/그래프가 읽기 쉬운가?
- [ ] 이미지 해상도가 적절한가?
- [ ] 오타가 없는가?
- [ ] 파일이 정상 작동하는가?
- [ ] 출력/공유 준비가 되었는가?
- [ ] 전체 리허설을 했는가?
- [ ] 시간을 재었는가?
- [ ] 핵심 메시지를 암기했는가?
- [ ] 전환 멘트를 준비했는가?
- [ ] 예상 질문 답을 준비했는가?
- [ ] 참석자 명단을 확인했는가?
- [ ] 회의실 환경을 확인했는가? (빔프로젝터 등)
- [ ] 백업 파일이 있는가? (USB, 이메일)
- [ ] 인쇄본이 필요한가?
- [ ] 후속 자료를 준비했는가?
