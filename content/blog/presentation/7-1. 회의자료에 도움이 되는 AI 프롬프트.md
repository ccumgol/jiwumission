>
> - 아래 템플릿에서 [대괄호]만 채워 넣고 그대로 붙여넣어 사용하세요.
> - 좋은 프롬프트의 핵심은 **역할 → 맥락 → 입력 → 출력 형식 → 제약/평가기준** 순서로 명확히 주는 것입니다.

## 프롬프트 1. 회의 자료 구조 설계 프롬프트

**역할(Role)**

- 당신은 회의 자료(슬라이드) 구조를 설계하는 프레젠테이션 코치입니다.

**목표/과업(Task)**

- 아래 정보를 바탕으로 회의 자료의 전체 구조를 설계해 주세요.

**맥락(Context)**

- 회의는 제한된 시간 안에 빠르게 이해하고 결정을 내리는 것이 목표입니다.

**입력(Input)**

- 회의 유형: [정기보고/의사결정/브레인스토밍/임원보고]
- 주제: [주제]
- 참석자: [직책/인원]
- 시간: [분]
- 목적: [목적]

**출력 형식(Output format)**

1. 슬라이드 목차 (번호 + 제목)
2. 각 슬라이드 핵심 메시지 1문장
3. 필요한 데이터/자료 목록
4. 시간 배분 (슬라이드별)
5. 예상 질문 5개

**제약/기준(Constraints)**

- 결론 우선 구조
- 1슬라이드 1메시지
- [15]분 이내 발표 가능하도록 간결하게

---

## 프롬프트 2. 데이터를 스토리로 바꾸는 프롬프트

**역할(Role)**

- 당신은 숫자 데이터를 설득력 있는 스토리와 슬라이드로 바꾸는 데이터 스토리텔러입니다.

**목표/과업(Task)**

- 아래 원본 데이터를 기반으로 “인사이트 중심” 슬라이드 구성을 제안해 주세요.

**입력(Input)**

- 원본 데이터: [원본 데이터 붙여넣기]

**출력 형식(Output format)**

1. 헤드라인 (insight 1문장)
2. 시각화 제안 (어떤 그래프가 적절한지)
3. 해석 (왜 중요한가)
4. 다음 액션 (무엇을 해야 하나)

**제약/기준(Constraints)**

- 긍정적 신호와 문제점을 모두 포함
- 구체적 숫자 포함
- 비교를 활용 (전월 대비, 작년 대비 등)

---

## 프롬프트 3. 옵션 비교표 생성 프롬프트

**역할(Role)**

- 당신은 의사결정을 돕는 분석가입니다.

**목표/과업(Task)**

- 선택지를 객관적으로 비교하고 추천안을 제시해 주세요.

**입력(Input)**

- Option A: [설명]
- Option B: [설명]
- Option C: [설명]
- 비교 기준:
  - [기준 1]
  - [기준 2]
  - [기준 3]

**출력 형식(Output format)**

1. 비교 표 (마크다운)
2. 각 옵션의 장단점 (각 3가지)
3. 점수화 (가능하면 가중치 적용)
4. 추천 의견 + 근거

**제약/기준(Constraints)**

- 가정이 있으면 “가정”으로 명시
- 가능한 한 정량화
- 결론(추천)을 맨 위 또는 마지막에 한 줄로 명확히

## 프롬프트 4. 회의록 → 다음 회의 자료 변환 프롬프트

**역할(Role)**

- 당신은 회의 운영을 돕는 프로젝트 매니저입니다.

**목표/과업(Task)**

- 이전 회의록을 바탕으로 다음 회의에서 바로 쓸 수 있는 슬라이드 구조를 만들어 주세요.

**입력(Input)**

- 이전 회의록:
  - 날짜: [날짜]
  - 결정사항: [결정사항]
  - Action Items: [액션아이템]
  - 미해결: [미해결]
- 다음 회의:
  - 날짜: [날짜]
  - 목적: [목적]

**출력 형식(Output format)**

- [슬라이드 1] 지난 회의 결정사항
- [슬라이드 2] 완료된 액션 (✅)
- [슬라이드 3] 진행 중인 액션 (🔄)
- [슬라이드 4] 지연된 액션 (🔴)
- [슬라이드 5] 이번 회의 결정사항

**제약/기준(Constraints)**

- 각 슬라이드당 bullet 3~5개로 제한
- 담당자(Who)와 기한(When)이 드러나게 작성

## 프롬프트 5. Executive Summary(임원용 1페이지 요약) 프롬프트

**역할(Role)**

- 당신은 임원 보고용 요약을 쓰는 커뮤니케이션 전문가입니다.

**목표/과업(Task)**

- 아래 상세 내용을 임원용 1페이지 요약으로 압축해 주세요.

**입력(Input)**

- 상세 내용: [상세 내용 10페이지 붙여넣기]

**출력 형식(Output format)**

- 📊 현황 요약 (3~4줄)
- 🎯 주요 성과 (3개, 숫자 포함)
- ⚠️ 주요 이슈 (2개)
- 💡 제안사항 (1~2개)
- 💰 예산 요청 (금액 + 근거 1줄)
- ✅ 승인 요청 (체크박스)

**제약/기준(Constraints)**

- 총 200단어 이내
- 숫자 중심
- 액션 명확
- 읽는데 2분
