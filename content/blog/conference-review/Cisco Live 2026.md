# Cisco Live US 2026 컨퍼런스 리뷰

**작성일**: 2026년 6월 6일 | **작성**: 시니어 테크 애널리스트

---

## 1. 행사 개요 (Executive Summary)

Cisco Live US 2026은 "에이전틱 AI 시대, 인프라가 스스로 운영하고 방어한다"는 단 하나의 주제로 수렴된 행사였다. 시스코는 분산된 콘솔과 도구들을 하나의 단일 플랫폼으로 통합한 **Cisco Cloud Control**을 전면에 내세우며, 2024년 Jeetu Patel CPO가 예고했던 "2년 후 시스코는 알아볼 수 없을 만큼 달라져 있을 것"이라는 약속을 공식적으로 이행했다. 이번 행사에서 시스코는 분기 사상 최대 매출($158억, 전년 대비 +12%) 달성이라는 강력한 재무적 배경 위에서, AI 인프라 연간 하이퍼스케일러 오더 목표를 기존 $50억에서 $90억으로 두 배 가까이 상향 조정하며 자신감 있는 로드맵을 제시했다.

**올해 행사의 한 줄 테마**: 에이전틱 운영(AgenticOps) — 인간과 AI 에이전트가 하나의 플랫폼에서 공동 작업하는 인프라 패러다임 전환

**가장 주목받은 발표 Top 3**:

1. **Cisco Cloud Control** — 네트워킹·보안·컴퓨팅·관찰가능성·협업을 단일 플랫폼에서 통합 운영
2. **Cisco Data Fabric (Splunk)** — 페더레이션 검색·머신 데이터 레이크·에이전틱 SOC로 구성된 텔레메트리 지능화 플랫폼
3. **퀀텀 안전(Quantum-Safe) 선언** — 2026년 12월까지 핵심 포트폴리오 대부분에 양자 안전 통신 적용 약속

**2025년 대비 가장 큰 전략적 전환**: 2025년이 Splunk 통합 성과를 가시화하는 해였다면, 2026년은 Catalyst·Meraki·Nexus·WebEx·Intersight·Splunk를 단일 로그인·단일 데이터 레이어·공통 에이전트 프레임워크로 묶는 플랫폼 회사로의 전환이 실체화된 해다. 특히 "단일 창(Single Pane of Glass)"이 아닌 "공유 데이터 레이어 위의 AgenticOps"라는 개념 전환이 핵심이다.

### 3-1. AI / 에이전틱 운영 플랫폼

---

**[Cisco Cloud Control]**

- **카테고리**: AI/ML, 플랫폼 관리
- **발표 내용**: 네트워킹(Catalyst, Meraki, Nexus), 보안, 컴퓨팅(Intersight), 관찰가능성(ThousandEyes, Splunk), 협업(Webex)을 단일 로그인·단일 데이터 레이어로 통합한 에이전틱 운영 플랫폼. 인간과 AI 에이전트가 동일한 운영 컨텍스트를 공유하며, 인간은 에이전트가 제안하는 액션의 최종 승인 권한을 보유(AgenticOps 모델).
- **출시 상태**: ✅ GA (2026년 6월 2일 발표 즉시, Cisco Live US 현장)
- **주요 구성 요소**:
  - **AI Canvas**: 운영자와 에이전트가 실시간 데이터를 공유하며 장애를 공동 조사·해결하는 멀티플레이어 워크스페이스
  - **Actions Queue**: 에이전트가 생성한 권고사항·근본 원인 분석·신뢰도 점수를 인간이 검토 후 실행하는 대기열
  - **Cloud Control Marketplace**: ServiceNow, Atlassian, BMC, Okta, Ping Identity, Microsoft Entra ID, Jamf, LiveAction, Panduit 등과 통합 (런칭 시점)
  - **Tokenomics 대시보드**: Galileo+Splunk 기반으로 AI 에이전트별 토큰 소비를 실시간 모니터링, 예산 초과 에이전트 즉시 종료 가능
- **출시 일정**: GA 즉시 / Cloud Control Studio는 2026년 말 예정
- **지원 리전**: 글로벌 (특정 리전 제한 미공개)
- **정량적 지표**: AI 에이전트는 동일 작업 대비 인간보다 450% 더 많은 네트워크 트래픽을 발생시킴 (Cisco 자체 연구, Newsroom 발표)
- **가격 정보**: 미공개 (Cisco 계정 담당자 문의 필요)
- **경쟁 제품**: HPE Aruba Central + Juniper Mist (통합 관리 플랫폼), Juniper Apstra (의도 기반 네트워킹)
- **출처**: https://newsroom.cisco.com/c/r/newsroom/en/us/a/y2026/m06/cisco-unveils-agentic-platform-for-operating-and-defending-critical-it-infrastructure.html

### 3-3. 데이터 / 관찰가능성 (Splunk)

---

**[Cisco Data Fabric powered by Splunk — 대규모 업데이트]**

- **카테고리**: DB/분석, 관찰가능성
- **발표 내용**: 2025년 9월 데뷔 이후 최대 규모 업데이트. 네트워크·애플리케이션·보안·서드파티 소스 텔레메트리를 공통 레이어로 통합, Cloud Control과 에이전틱 SOC의 데이터 기반으로 역할.
  - **Federated Search (확장)**: 데이터를 Splunk로 이동시키지 않고 Splunk를 데이터로 이동시키는 방식, SPL2 통합 쿼리 언어로 분산 환경 단일 검색. 비용·성능 효율 최대 10배 향상 (Splunk 공식 발표)
  - **Turnkey Machine Data Lake**: 원시 머신 데이터 스키마 자동 관리, 수동 ETL 작업 불필요
  - **AI Toolkit Agent Builder**: 노코드 인터페이스로 완전 자율 에이전트 빌드·배포
  - **Agentic SOC**: 6개 전용 보안 에이전트로 위협 탐지·근본 원인 분석·사고 대응 자동화. Triage Agent, Detection Studio, SOP Agent 등 포함.
  - **Splunk Agent Observability (Galileo 기반)**: AI 에이전트 전체 개발 생애주기 모니터링, 의도치 않은 행동·유해 출력 차단, 비용 제어
  - **AI SRE**: Splunk Observability Cloud에서 자동 근본 원인 분석 및 수정 가이드 제공

## 8. 타깃 독자별 핵심 요약

### 경영진/의사결정자가 알아야 할 3가지

1. **AI 인프라 투자는 수익으로 이어지고 있다**: 시스코 Q3 FY2026 분기 최고 매출 $158억, 제품 오더 +35% 성장은 AI 인프라 수요가 마케팅이 아닌 실수요임을 입증한다. 하이퍼스케일러 AI 인프라 오더만 FY2026 $90억 목표로 전년 대비 약 4.5배 성장했다. 우리 조직의 AI 인프라 예산이 경쟁사에 비해 지연되고 있다면 이를 긴급하게 재검토해야 할 시점이다.
2. **에이전틱 AI의 총소유비용(TCO)은 '토크노믹스' 거버넌스 없이는 통제 불가**: AI 에이전트 1개가 권한 범위 제한 없이 자율 실행되면 수일 내에 조직 AI 예산의 상당 부분을 소진할 수 있다는 실제 리스크가 부상했다. Cloud Control의 Tokenomics 대시보드는 이를 실시간으로 통제하는 필수 거버넌스 레이어다. 에이전틱 AI 파일럿을 승인하기 전에 반드시 토큰 비용 거버넌스 정책을 수립해야 한다.
3. **양자 컴퓨팅 대응은 지금 시작해야 한다**: 시스코가 2026년 12월 핵심 장비에 양자 안전 기본 탑재를 선언한 것은 "harvest now, decrypt later" 공격(현재 데이터를 암호화된 채로 수집해 미래에 복호화)이 이미 현실의 위협임을 의미한다. 현재 암호화 인프라 및 민감 데이터 자산에 대한 양자 리스크 평가를 2026년 내 완료하는 것을 권장한다.

---

### 아키텍트가 알아야 할 3가지

1. **Cloud Control은 "단일 창"이 아닌 "공유 데이터 레이어"다**: 과거 통합 관리 플랫폼이 단순히 여러 콘솔을 탭으로 묶은 것이었다면, Cloud Control은 Catalyst·Meraki·Nexus·Webex·Intersight·Splunk가 동일한 데이터 레이어를 공유한다. 기존 개별 도메인 전문팀 사일로 구조를 재검토하고, 크로스 도메인 인시던트 대응 워크플로우를 AgenticOps 모델로 재설계할 필요가 있다.
2. **Hybrid Mesh Firewall = 방화벽 어플라이언스 아키텍처의 종말의 시작**: Nexus 9300/Catalyst 9350 Smart Switch에 스테이트풀 세그멘테이션이 와이어 속도로 내장된다는 것은, AI 데이터센터와 캠퍼스 모두에서 별도 방화벽 어플라이언스 없이 정책 적용이 가능한 시대가 열린다는 의미다. 신규 데이터센터·캠퍼스 설계 시 어플라이언스 중심 아키텍처를 패브릭 내장 보안으로 전환하는 방향을 검토해야 한다.
3. **Digital Twin + Federated Search가 인프라 변경 관리를 바꾼다**: Digital Twin(실제 소프트웨어 이미지 에뮬레이션, 2026년 7월 알파)으로 프로덕션 영향 없이 변경 사항을 사전 검증하고, Federated Search로 데이터를 이동시키지 않고 분산 환경을 단일 쿼리로 조사할 수 있다. 이는 ITSM(IT 서비스 관리)의 CAB(변경 자문 위원회) 프로세스를 AI 에이전트 주도로 재편하는 기반이 된다.

---

### 개발자가 알아야 할 3가지

1. **DefenseClaw를 지금 살펴볼 것**: Claude Code, Codex, Cursor, Gemini CLI 등 AI 코딩 도구를 이미 사용하고 있다면, DefenseClaw 오픈소스 거버넌스 레이어를 통해 해당 에이전트가 기업 시스템에 접근할 때 적용되는 Zero Trust 제어를 이해해야 한다. 기업 내 에이전틱 AI 개발 표준 정책의 출발점으로 활용 가능하다.
2. **Cloud Control Studio (2026년 말) 출시 전 MCP 역량을 미리 확보할 것**: Cloud Control Studio의 Agent Builder는 MCP(Model Context Protocol)로 50개 이상 서드파티 플랫폼에 연결된다. 시스코 인프라 위에서 커스텀 에이전트를 빌드할 계획이라면, MCP 서버 개발 역량과 프롬프트 엔지니어링 기술을 선제적으로 익혀두는 것이 좋다.
3. **CCNA v2.0 대비 조기 학습 기회를 활용할 것**: Cisco U에서 CCNA 업데이트 공식 트레이닝이 무료로 조기 공개됐다. 새 시험은 2027년 2월부터 유효하며, 기존 v1.1과 동시 적용 기간이 있다. AI 도구가 통합된 CCIE Practical 시험 변경도 네트워크 엔지니어에게 중요한 변화다.

- **CCNA 업데이트 트레이닝 (Cisco U)**: 네트워크 엔지니어팀을 위한 무료 조기 공개 커리큘럼 즉시 활용.

### 🟡 신청/대기 등록이 필요한 프리뷰 기능

- **Cisco Multicloud Fabric 베타 (2026년 6월)**: AWS/Azure/GCP 멀티클라우드 연결을 단일 패브릭으로 운영하는 NaaS 베타 참가 신청.
- **AgenticOps Agentic Actions (Meraki 베타, 2026년 6월)**: 자율 네트워크 장애 감지·수정 5단계 루프 실사용 가능. Meraki 대시보드 관리 환경 필수.
- **Digital Twin 알파 (2026년 7월)**: 실제 소프트웨어 이미지 기반 네트워크 에뮬레이션. 변경 관리 프로세스 혁신 희망 기업 우선 지원.
- **Quantum Ready Assessments (Cisco IQ, 2026년 7월 GA)**: 자사 암호화 자산의 양자 리스크 우선순위 평가. 금융·공공기관은 조기 진입 권장.

### 🔄 기존 아키텍처 재검토가 필요한 영역

- **방화벽 어플라이언스 vs. Smart Switch 내장 보안**: Nexus 9300 및 Catalyst 9350의 스테이트풀 세그멘테이션 기능이 성숙되면 별도 어플라이언스 불필요 시나리오 등장. 3~5년 내 데이터센터/캠퍼스 방화벽 아키텍처 재설계 로드맵 준비 필요.
- **AI 에이전트 ID 관리 및 권한 범위(Scope) 거버넌스**: AI 에이전트를 인간 사용자와 동일한 IAM(ID 및 접근 관리) 프레임워크에 통합. Astrix + Cisco Identity Intelligence 통합 방향을 주시하며 기존 ITAM/IAM 정책 검토.
- **네트워크 업스트림↔다운스트림 트래픽 비율 재설계**: 에이전트 트래픽은 기존 인간 주도 트래픽과 달리 업스트림(서버→클라이언트) 비율이 역전될 수 있음. WAN 및 코어 스위치 용량 계획 시 에이전트 트래픽 비율을 별도로 모델링 필요.

### 📚 추가 학습/리서치가 필요한 주제

- **MCP (Model Context Protocol)**: Cloud Control Studio Agent Builder의 핵심 통신 프로토콜. 50개 이상 서드파티 연동의 기반.
- **eBPF 기반 보안 (Isovalent)**: Live Protect의 기반 기술. 재부팅 없는 런타임 보안 패치의 원리 이해에 필수.
- **AgenticOps 조직 변화 관리**: AI 에이전트가 운영 1등 시민이 되면 NOC(네트워크 운영 센터)/SOC(보안 운영 센터)의 역할과 필요 인력 프로파일이 변화. 인력 재교육 계획 수립 필요.
- **Post-Quantum Cryptography (PQC)**: NIST PQC 표준(ML-KEM, ML-DSA 등)과 Cisco 양자 안전 로드맵의 교차점 이해.
